# 图片生成计划

## 适用场景

当需要把故事、剧本、镜头组、制作包、漫画章节或已有资产，整理成一份“后续逐张生成图片”的计划文档时，使用本技能。

本技能只规定计划文档的格式、写法、信息完整度和约束纪律，不绑定任何具体项目画风。

## 核心原则

1. **只写计划，不生成图片**
   - 文档只包含生成顺序、保存路径、参考图、提示词和检查项。
   - 不调用图片生成工具，不提交任务，不把“待生成”误写成“已生成”。

2. **计划服从源文档**
   - 图片计划必须服务已有故事、剧本、镜头组或制作包。
   - 不借图片计划新增剧情、改写人物关系、调换镜头顺序或补不存在的设定。
   - 源文档冲突时，先标注冲突或等待确认，不在图片计划里自行裁决。

3. **镜头组是关键帧来源，不是单镜头表**
   - 每张剧情关键帧优先对应一个镜头组，必要时多个镜头组合并为一张关键帧。
   - 不默认为组内每个节拍生成一张图；只有身份、空间、道具或动作结果明显不同时才拆成多张。
   - 漫画页和视频关键帧也按镜头组组织，避免把长故事直接变成零散单图。

4. **先资产，后关键帧**
   - 先生成身份锚点和基础资产。
   - 再生成场景、道具、服装、状态参考。
   - 最后生成剧情关键帧、漫画页或视频关键帧。
   - 后面的图片需要引用前面的图片时，先把前序图片写入计划并给出保存路径。

5. **路径、参考图、提示词必须可执行**
   - 每张图都有明确保存路径。
   - 参考图必须写明来源和用途。
   - 提示词必须能直接交给图片生成工具，而不是只写剧情摘要。

6. **约束写在计划里，不靠记忆执行**
   - 人物身份、人数、禁用素材、尺度边界、参考图用途、避免项都必须写清楚。

## 推荐文档结构

```markdown
# 《项目 / 集数 / 章节》图片生成计划

> 目标：为 [源文档](相对路径) 整理后续逐张生成图片的计划。
> 本文档只写生成计划、保存路径、提示词、参考图；**不生成图片**。
> 本计划只服务当前源文档；**不修改故事、不修改镜头组/制作包**。

## 0. 本次图片生成总规则

### 0.1 源文档与范围
### 0.2 固定参考资产 / 美术规则
### 0.3 角色、人数与连续性规则
### 0.4 尺度、禁用内容与特殊注意事项

## 1. 生成顺序总览

## 2. 角色 / 状态参考图

## 3. 场景 / 环境图

## 4. 道具 / 服装 / 物件参考图

## 5. 剧情关键帧 / 漫画页 / 视频关键帧

## 6. 生成前检查清单
```

章节可按项目需要增删，但应保持“总规则 → 顺序总览 → 分类型图片计划 → 检查清单”的顺序。

## 总规则写法

### 源文档与范围

必须说明：

- 计划依据哪些文件：故事、剧本、镜头组、制作包、角色设定、资产索引等。
- 本计划覆盖的集数、章节、场景范围。
- 哪些旧设定、废弃目录、未采用素材、过时剧情不能使用。
- 图片计划不得反向改写剧情或镜头组。

### 固定参考资产 / 美术规则

必须说明：

- 固定参考图有哪些。
- 每张参考图用于什么：身份、服装、场景、光感、构图、材质、情绪等。
- 哪些内容只参考风格，不能复制具体物件或角色。
- 后续提示词采用什么统一公式。

通用提示词公式：

```text
【基础美术风格】<只写画面语言、风格、材质、光影、细节密度等通用规则>
【具体生成内容】<写主体、动作、地点、道具、状态、人数、构图目的>
【避免】<写本图最关键的禁项；不要堆砌无关负面词>
```

注意：

- `【基础美术风格】` 不应混入具体人物、剧情、道具或项目专属设定。
- `【具体生成内容】` 承担主体、动作、地点和剧情信息。
- `【避免】` 只写会显著影响生成质量或连续性的禁项。

### 角色、人数与连续性规则

必须说明：

- 每个主要角色的身份锚点参考图。
- 哪些图必须保持同一张脸、同一服装、同一道具或同一场景结构。
- 同一画面内能出现几个命名角色。
- 是否禁止同一人物重复出现、残影式连续动作、自动加人。
- 群演、路人、士兵、怪物等是否有精确数量要求。

## 单张图片条目格式

```markdown
### <编号> <图片名称>

- **保存路径**：[待生成：文件名.png](相对/保存路径/文件名.png)
- **类型**：<角色状态参考 / 纯环境 / 道具参考 / 概念剧照 / 漫画页 / 视频关键帧>
- **建议画幅**：<横图 16:9 / 竖图 2:3 / 横图 4:3 / 方图 1:1 / 按项目需要>
- **参考图**：
  - 图1：[参考图文件名](相对路径) — <只写该参考图用于什么>
  - 图2：[参考图文件名](相对路径) — <只写该参考图用于什么>
- **提示词**：

```text
【基础美术风格】……
【具体生成内容】……
【避免】……
```
```

## 编号规则

推荐使用稳定编号：

- `CH-01`：角色 / 状态参考图。
- `ENV-01`：环境 / 场景空镜。
- `PROP-01`：道具 / 服装 / 物件。
- `KF-01` 或 `STILL-01`：由镜头组导出的剧情关键帧 / 概念剧照。
- `PAGE-01`：漫画页 / 分格页。

编号一旦写入文档，后续尽量不要重排；取消时保留条目并标注“取消”。

## 提示词重点

### 角色图

- 身份参考图。
- 服装 / 状态参考图。
- 完整全身、半身、头肩、干净背景或简化背景。
- 姿态、表情、状态。
- 避免换脸、重复人物、错误服装、过度复杂配件。

### 环境图

- 无人物或无命名角色。
- 地点、时间、空间结构。
- 后续动作需要的站位、路线、入口、遮挡、道具位置。
- 避免空间不连贯、主体误入、文字/Logo/水印、细节过满。

### 道具图

- 道具数量和摆放方式。
- 尺度、材质、结构可读性。
- 是否需要展示打开、闭合、分解或组合状态。
- 避免品牌文字、玩具感、结构不可读。

### 关键帧

- 谁在场。
- 在哪里。
- 正在做什么。
- 危险或冲突从哪里来。
- 对应源文档哪一段 / 哪个镜头组。
- 为什么这一组只需要一张关键帧，或为什么必须拆成多张。
- 人数和角色不要自动增减。

## 取消项写法

如果某张图不再生成，不要直接删除。保留条目，明确状态、原因和替代处理，避免后续误补。

```markdown
### CH-05 某角色参考图（取消）

- **状态**：取消生成，不再作为角色图或参考图。
- **原因**：该角色只作为伏笔存在，当前章节不提前给出明确形象。
- **处理方式**：后续关键帧只用环境、声音、角色反应或不可辨认剪影暗示；禁止画出完整可识别形象。
```

## 检查清单

- [ ] 是否只依据当前源文档，没有引入旧剧情或废弃设定。
- [ ] 是否每张图都有保存路径、类型、建议画幅、参考图和提示词。
- [ ] 是否所有路径都是相对路径，能从本文档位置正确跳转。
- [ ] 是否明确哪些图是“待生成”，哪些图是“可复用”。
- [ ] 是否先生成身份锚点、场景、道具，再生成关键帧。
- [ ] 是否按镜头组导出关键帧，而不是为每个单镜头机械生成。
- [ ] 是否每张参考图都说明了具体参考用途。
- [ ] 是否提示词包含 `【基础美术风格】`、`【具体生成内容】` 和必要的 `【避免】`。
- [ ] 是否人物数量、命名角色数量、重复人物规则写清楚。
- [ ] 是否没有让图片计划反向改写故事、剧本或镜头组。
- [ ] 是否没有提交图片生成任务；本文档只是计划。
