# 图片生成计划

## 适用场景

当需要把故事、剧本、镜头组、制作包、漫画章节或已有资产，整理成一份“后续逐张生成图片”的 TOML 计划文件时，使用本技能。

本技能只规定 TOML 计划文件的格式、写法、信息完整度和约束纪律，不绑定任何具体项目画风。

## 核心原则

1. **只写计划，不生成图片**
   - TOML 文件只包含 comments、任务数组、保存路径、参考图、画幅和提示词。
   - 不调用图片生成工具，不提交任务，不把“待生成”误写成“已生成”。

2. **图片生成计划必须等镜头组之后再做**
   - 图片生成计划不能从剧本直接跳过来生成；必须先有镜头组 / 分镜节拍 / shot table 作为直接依据。
   - 如果用户只提供剧本、故事或分集大纲，却要求直接制作图片生成计划，必须先打断并提醒：需要先完成镜头组；不能为了省步骤直接写图片计划。
   - 只有当镜头组已经存在，或用户明确提供等价的镜头组文档时，才继续制作图片生成计划。

3. **计划服从源文档**
   - 图片计划必须服务已有镜头组，并回溯对应故事、剧本或制作包。
   - 不借图片计划新增剧情、改写人物关系、调换镜头顺序或补不存在的设定。
   - 源文档冲突时，先标注冲突或等待确认，不在图片计划里自行裁决。

4. **镜头组是概念剧照来源，不是单镜头表**
   - 每张剧情概念剧照优先对应一个镜头组，必要时多个镜头组合并为一张概念剧照。概念剧照不是最终成片中 exactly 出现的 frame，而是服务人物、场景、情绪、动作关系和视频生成参考的宽用途剧照。
   - 不默认为组内每个节拍生成一张图；只有身份、空间、道具或动作结果明显不同时才拆成多张。
   - 漫画页和视频概念剧照也按镜头组组织，避免把长故事直接变成零散单图。

4. **角色和道具中心管理，先复用后新增**
   - 长篇 / 多集项目中，跨集角色的脸、年龄段、服装状态和立绘属于项目级角色资产，放在 `人物/<角色名>/` 下，不跟随单集目录反复生成；同一角色的图片直接放在同一个角色文件夹内，不再按 `源头大头照/`、`立绘/` 等拆多层。
   - 长篇 / 多集项目中，跨集使用的重要道具属于项目级道具资产，放在项目级 `道具/` 目录；如果剧情上是同一个道具（如锡杖、行囊、经卷、念珠、书箱、马鞍等），只能生成一张主参考图，后续各集只引用它，不要在每集 `资产/EPxx/道具/` 下重复生成同名道具。
   - 单集目录主要保存本集场景、本集一次性道具、概念剧照、镜头组和图片生成计划；单集计划引用项目级人物与项目级道具资产。
   - 写计划前必须先检查已有 `人物/`、`道具/`、前集资产索引和已生成图片；能复用的角色/道具先复用，只有剧情明确出现新道具、道具外观永久变化、损坏/打开/特殊状态需要单独表现时，才新增道具图。
   - 强制生成顺序：按剧集批次推进；每个计划文件 / 批次内固定为 1) 环境 / 场景，2) objects / tools / props / clothes / costumes 等非人物物件资产，3) characters / roles / 人物身份、脸部、立绘、状态图，4) conceptual scene shots / 概念剧照。服装、衣物、道具、工具都必须排在人物之前；概念剧照永远最后。
   - 后面的图片需要引用前面的图片时，先把前序图片写入计划并给出保存路径。

5. **跨集角色只定一次源头脸**
   - 角色第一次出现时生成并确定唯一源头大头照 / 正脸图，作为项目级身份锚点。源头脸重点定义脸、头型、发式、五官、脖颈和少量肩部轮廓，不承担完整服装、道具、场景或氛围定义。
   - 源头大头照裁切应为近距离身份头像：画面从头顶到锁骨下方少量肩部；允许少量符合身份的衣领 / 上衣质感辅助判断时代与身份。
   - 源头大头照背景使用柔和虚化的低细节历史环境背景，背景服务角色气质和时代感，人物脸部必须清晰突出。不要使用纯白背景做成过硬的证件照质感。
   - 如果已有源头大头照露出大面积肩胸服装、复杂服饰层级、显眼道具、法器、佛珠或清晰剧情背景，必须判为不合格，并将对应源头脸任务标为 `# todo` 重新生成。
   - 后续集数不再为同一年龄阶段重复生成该角色大头照；只在服装、装备、剧情状态变化时补服装图和人物立绘。
   - 如果剧情跨越多年，可按关键年龄阶段生成新的年龄脸；所有年龄脸以及所有带有人物脸部的图片，都必须且只能参考源头脸来锁身份，不能参考上一阶段脸、剧照或立绘来传递身份。

6. **路径、参考图、提示词必须可执行**
   - 每张图都有明确保存路径。
   - 参考图必须写明来源和用途。
   - 提示词必须能直接交给图片生成工具，而不是只写剧情摘要。

7. **约束写在计划里，不靠记忆执行**
   - 人物身份、人数、禁用素材、尺度边界、参考图用途、避免项都必须写清楚。

8. **长篇计划默认用一层批次 TOML，不按资产类型拆计划目录**
   - 多集项目的 `图片生成计划/` 目录默认保持一层：`1.toml`、`2.toml`、`3.toml`……；每个 TOML 对应一个分集 / 批次。
   - TOML 文件名数字顺序就是总执行顺序；单个 TOML 内 `[[tasks]]` 的自然顺序就是本批次生成顺序。不要再额外维护 `00_制作顺序/`、`制作顺序.md` 或任务索引文档。
   - 不要把计划目录拆成 `01_主人物/`、`02_配角/`、`03_环境/`、`04_服装道具/`、`05_分集/`。这些是资产分类思路，不是生成计划组织方式。
   - 计划批次与最终资产目录不需要保持同一种队形：任务按剧情批次执行，但 `target_image_path` 仍可指向 `资产/人物/`、`资产/道具/`、`资产/EPxx/场景/`、`资产/EPxx/概念剧照/` 等解耦保存路径。
   - 扁平化或重排后必须验证：一层目录无子文件夹、批次 TOML 数量正确、所有 TOML 可解析、任务总数无丢失、ID 唯一、每个任务必需字段完整、所有 `target_image_path` 仍指向 `资产/` 下的正确解耦路径。

## TOML 文件结构总览

输出必须是 `.toml` 文件，不输出 TOML 计划正文，不使用 TOML 标题、引用块、勾选清单或单张图片条目模板。所有需要写给 agent 看的额外说明，写成 TOML comments：

```toml
# 《项目 / 集数 / 章节》图片生成计划
# 依据：镜头组_42集版/EPxx_xxx_镜头组.md；对应剧本：剧本_42集版/EPxx_xxx.md。
# agent执行规则：本计划用于agent模型生成图片；每一次生成时，传给agent的 prompt、reference_image_paths、aspect_ratio 和其他参数，必须严格遵照当前单独的 [[tasks]]。
# 任务顺序：每个计划文件内严格按 1) ENV/environment，2) objects/tools/props/clothes/costumes，3) CH/characters/roles，4) STILL/conceptual scene shots 排序。

[[tasks]]
id = "ENV-01"
target_image_path = "资产/EPxx/场景/S01_主空间.png" # todo
reference_image_paths = []
aspect_ratio = "16:9"
prompt = """..."""
```

TOML comments 的用途：记录源文档、范围、全局执行规则、生成顺序、重要资产约束和自检提示。Comments 只能辅助执行，不能替代任何单个 `[[tasks]]` 的字段；每个任务仍必须 self-contained。

## TOML comments 写法

### 源文档与范围 comments

必须用 `#` comments 说明：

- 计划依据哪些镜头组文件；镜头组对应哪些故事、剧本、制作包、角色设定、资产索引等。
- 本计划覆盖的集数、章节、镜头组范围。
- 如果没有镜头组，停止制作图片生成计划，先提醒用户补做镜头组。
- 哪些旧设定、废弃目录、未采用素材、过时剧情不能使用。
- 图片计划不得反向改写剧情或镜头组。

### 固定参考资产 / 美术规则 comments

必须用 `#` comments 说明：

- 固定参考图有哪些。
- 每张参考图用于什么：身份、服装、场景、光感、构图、材质、情绪等。
- 哪些内容只参考风格，不能复制具体物件或角色。
- 后续提示词采用什么统一公式。
- 如果项目有明确的 base style 文件（如 `base-art-style.md`、`base-style.md`、`art references/base-art-style.md`），该文件就是基础画风的硬约束：提示词里的基础美术行必须逐字采用该文件内容或明确引用该文件，不得写近似改写版；提示词其他部分也不得加入与该文件媒介、渲染、材质、光影、色彩、对比度或细节密度相冲突的词语。
- 审查既有 TOML 时，必须主动指出并改写；只要改动 prompt/reference/aspect/关键约束，即使图片已存在，也应把对应任务标为 `# todo`。

通用提示词公式：

```text
【基础美术风格】<电影项目只写硬摄影规格：摄影机型号、记录格式、镜头品牌/系列/焦段、T 值、画幅比例；保持为摄影部门语言>
【具体生成内容】<写主体、动作、地点、道具、状态、人数、构图目的>
【生成要求】<用正向语言写清本图必须满足的构图、身份、时代、画幅、背景、材质和参考图约束>
```

注意：

- `【基础美术风格】` 不应混入具体人物、剧情、道具或项目专属设定。
- 电影 / 短剧 / 影视项目的 `【基础美术风格】` 应采用摄影部门语言，而不是抽象美术口号。必须明确摄影机和镜头型号，例如 `ARRI ALEXA 35 + Cooke Anamorphic/i 40mm`、`Sony VENICE 2 + Panavision Primo 50mm` 等；具体型号应服从项目统一摄影方案。
- `【基础美术风格】` 只放摄影机、记录格式、镜头系列/焦段、T 值、画幅比例等硬规格；不要写“安全构图、固定机位、光源依据、背景只交代空间、不使用现代柔光箱效果”等调度/说明文字，这些如必须出现，应放入具体生成内容且保持简短。
- 提示词是给没有上下文的图像模型（如 image2）使用的，必须 self-contained：`【提示词】` 内不能写 `对应镜头组 EPxx-SGxx`、`本图只选择`、`如上文`、`本集`、`镜头组限定` 等依赖外部文档的描述；每条提示词必须直接说明地点、人物、动作、道具、画面边界和禁项。
- 禁止把这类句子当作基础风格：`电影感写实历史纪录片风格，唐代与西域佛教题材，克制构图，真实材质，低饱和暖冷对比，自然尘埃与体积光，空间真实可行，细节清晰但不过度堆砌，无文字无水印。` 其中“真实材质、低饱和暖冷对比、自然尘埃与体积光、空间真实可行、细节清晰但不过度堆砌”等属于文学化/说明书化描述，应删掉或转化为具体摄影、灯光、镜头和场面调度信息。
- `【具体生成内容】` 承担主体、动作、地点和剧情信息；必须短句化、可直接投喂，不写制作备注或索引说明。
- 概念剧照是单帧图片，不是视频脚本。概念剧照提示词只写画面能直接看见的视觉信息：场景/地点、景别、构图、人物站位、人物神情、动作、可见道具和明确人数。不要写项目名、片名、`概念剧照`、`画面表现`、剧情说明、台词、长段故事前因后果，尤其不要把一段镜头组正文直接塞进提示词。
- 如果人物长相、服装或场景已经由参考图提供，prompt 内不要重复描述这些已有细节；只写 `图1用于场景空间和动线，图2用于玄奘身份与服装，图3用于麹文泰身份与服装` 这类用途，并补充本张图特有的站位、表情、动作、景别变化。
- 同一集相邻概念剧照必须主动设计差异：至少改变景别、镜头角度、人物距离/站位、动作状态或可见道具焦点之一。不能只替换台词或剧情段落而保持同样“两人对站说话”的画面。
- 判断是否需要拆成多张概念剧照的硬标准：如果两条任务的 `reference_image_paths` 完全相同，说明环境相同、主要人物相同；除非两张图的动作、事件、人物关系、景别或镜头角度非常不一样，否则不要写第二条，应合并成一条或删掉后者。只靠不同对白、不同剧情说明、轻微表情变化，不能成立为第二张图。
- 提示词中不使用 negative prompt。大原则：如果不想让图片里出现某物，就不要在提示词里提到它，哪怕是在 `【生成要求】`、negative prompt 或“不要/不得/禁止”句式里。改用正向约束，例如“纯白背景、单人正面头肩像、只呈现本任务角色、画面干净”。
-agent没有项目上下文。每个 `[[tasks]]` 的 prompt 必须 self-contained：直接写出画面里出现的主体、地点、动作、人数、道具、构图、光线、情绪和画面边界；不写“服务《某集》”“本段叙事”“源文档限定”“按画面需要”等依赖外部理解的句子。
- 如果任务需要参考图，`reference_image_paths` 必须完整，并在 prompt 中用 `图1`、`图2` 的方式说明每张参考图的用途。不要在 prompt 里重复叙述参考图已经包含的细节，只写“图1用于空间结构 / 图2用于人物身份 / 图3用于道具形制”等用途。
- 使用已有参考图前必须实际读图判断画面内容，不能只看文件名推断。若参考图尚未生成，则该参考图任务本身的 prompt 必须先把画面定义完整；下游任务引用它时只写用途。
- 若当前项目的图片 / art / visual style 不是写实照片风格，挑选既有参考图时优先使用文件名带 `styled` 前缀或后缀的版本；例如同时有 `ballroom_16x9_styled` 和 `ballroom_16x9` 时，优先选 `ballroom_16x9_styled`。没有 styled 版本时才使用普通版本。
- 重要场景需要先生成场景参考图，必要时拆成多张：主空间、局部行动区、特殊光效 / 梦境 / 仪式状态。概念剧照应引用这些场景参考图，而不是凭空描述大空间。

### 角色、人数与连续性 comments

必须用 `#` comments 说明：

- 每个主要角色的身份锚点参考图。
- 哪些图必须保持同一张脸、同一服装、同一道具或同一场景结构。
- 同一画面内能出现几个命名角色。
- 是否禁止同一人物重复出现、残影式连续动作、自动加人。
- 群演、路人、士兵、怪物等是否有精确数量要求。


## 剧集批次生成顺序与资产目录解耦

长篇 / 多集项目的图片生成计划应按分集批次组织成一层 TOML 文件，而不是按最终资产目录机械拆分。推荐结构：

```text
图片生成计划/
  README.md
  1.toml
  2.toml
  3.toml
  ...
```

规则：

- **文件名数字顺序就是总执行顺序**：`1.toml` → `2.toml` → `3.toml`……；不要再写 `00_制作顺序/`、`制作顺序.md` 或单独索引文档。
- **单个 TOML 内自然顺序就是本批执行顺序**：执行器直接按文件内 `[[tasks]]` 出现顺序生成；因此每个计划文件必须严格按 1) ENV/environment，2) objects/tools/props/clothes/costumes，3) CH/characters/roles，4) STILL/conceptual scene shots 排序。不要根据最终保存目录重排，也不要把 characters 放到 clothes/props/tools 之前。
- **计划目录不按资产类型解耦**：不要把 `图片生成计划/` 拆成 `01_主人物/`、`02_配角/`、`03_环境/`、`04_服装道具/`、`05_分集/`。这些会让执行顺序和保存目录混在一起，增加维护成本。
- **资产保存路径仍然解耦**：`target_image_path` 可以且应该按最终资产库组织：人物放 `资产/人物/`，道具放 `资产/道具/`，分集场景放 `资产/EPxx/场景/`，分集概念剧照放 `资产/EPxx/概念剧照/`。
- **任务归属按剧情首次需要出现的批次，文件内顺序按资产管线**：如果人物 A 第一次在 EP01 登场，则 A 的服装、源头脸和必要立绘归入 `1.toml`；如果人物 B 第一次在 EP04 登场，则归入 `4.toml`。但在任何一个文件内部，仍必须先写环境，再写 objects/tools/props/clothes/costumes，再写 characters/roles，最后写 conceptual scene shots。
- **年龄 / 状态变化跟随发生集数**：角色新年龄、新状态、新服装在剧情首次需要的批次出现；不要提前在“人物总表”里一次性全做完。
- **跨集复用资产仍放项目级路径**：同一剧情道具只生成一次主参考图，保存到 `资产/道具/`；后续批次只引用它。跨集角色源头脸、状态脸和立绘集中保存到 `资产/人物/<角色名>/`。
- **批次内强制顺序**：环境 / 场景 → objects / tools / props / clothes / costumes → characters / roles / 人物身份资产 → conceptual scene shots / 概念剧照。这个顺序必须直接体现在同一个 `N.toml` 内的 `[[tasks]]` 自然顺序中；服装/衣物属于第 2 类，必须排在人物前；概念剧照必须最后。
- **复检仍按生成顺序做**：每生成一张人物图后，按“上一张已生成的人物图 / 本批已通过人物图”做相似度复检；跨批次也延续记录。

这种规则模拟实拍制片流程：按拍摄 / 剧情推进批次落实场地、服化道和演员状态；但资产保存仍保持制片资产库结构，便于跨集复用。

## 源文档对照与未登场内容清理

当用户要求“对照剧本 / 镜头组删除从来没有登场的内容”时，必须先做源文档盘点，再改 TOML；不要凭角色名直觉删除。

流程：

1. **建立源文档语料**：读取对应 `剧本_*` 和 `镜头组_*` 目录内所有相关 `.md`，合并为可检索文本。
2. **解析计划任务**：用 TOML 解析每个 `N.toml`，提取 `id`、`target_image_path`、`reference_image_paths`、`prompt`，并从 `资产/人物/<角色名>/`、`资产/道具/<道具名>.png`、任务 id 中抽取实体名。
3. **按源文档称呼核对**：实体名必须能在剧本或镜头组中找到相同称呼，或找到明确同义称呼。若计划使用了源文档没有的别名（例如源文档写“少年玄奘”，计划写人物资产“陈祎”），优先改回源文档称呼，而不是删除剧情需要的人物。
4. **删除条件要严格**：只有实体及其同义称呼在剧本和镜头组中都没有出现，且没有被任何仍保留任务引用，才删除对应 `[[tasks]]`；如果它是后续任务参考图，先判断后续任务是否也应删除或改引用。
5. **不要删剧情需要的状态资产**：青年 / 中年 / 老年状态脸、服装图、项目级道具图即使不逐字出现在某一集剧本里，只要对应角色或道具在镜头组中出现，且该资产服务连续性，就保留或改名，不当作“未登场”。
6. **改名要同步链路**：改 `id`、`target_image_path`、`reference_image_paths`、prompt 内角色称呼和相关 `资产/人物/<角色名>/` 路径，避免旧称呼残留。
7. **验证**：修改后检查所有 TOML 可解析、ID 唯一、引用路径仍能对应到已保留目标或既有资产、被清理的未登场 token 在图片计划中为 0、所有人物资产目录名都能在源文档或同义映射中找到来源。

## TOML 图片生成计划格式

当用户指定使用 TOML 计划时，输出文件使用 `图片生成计划.toml` 或带集数/版本的同类文件名，根结构为任务数组：

TOML 文件开头必须写明agent执行规则：本计划用于agent模型生成图片；每一次生成时，传给agent的提示词、参考图和其他参数，一定要严格遵照当前单独的 `[[tasks]]`，不能根据上下文、相邻任务、既有图片或角色印象自行修改、补全、串用参考图或沿用其他任务的人脸。

TOML 的 `prompt` 必须只使用正向提示词，不写 negative prompt；不要用“不要、不得、禁止、避免、不能、不可、无、没有”等方式列出排除项，也不要在负面句里提到不想出现的对象。把排除意图改写为正向画面约束。

每个 TOML 任务都必须独立可执行：参考图列表完整，prompt 说明 exact 画面内容；如果引用参考图，按 `图1`、`图2` 标注用途。生成计划不能假设agent知道剧本、镜头组、上下文或前后任务。

```toml
[[tasks]]
id = "CH-少年玄奘-FACE"
target_image_path = "资产/人物/少年玄奘/少年玄奘_源头脸部特写_3x4.png" # todo
reference_image_paths = []
aspect_ratio = "3:4"
prompt = """【基础美术】ARRI ALEXA 35，Open Gate 4.6K，Cooke Anamorphic/i 40mm，T2.8，2.39:1
【具体内容】少年玄奘源头脸部特写，视觉年龄8岁左右的男童，正面头肩像，干净中性背景，五官清晰，神情聪敏、恭敬、认真到近乎执拗，唐代士族儿童衣冠气质，作为后续同一张脸的身份锚点。
【生成要求】纯白背景，单人正面头肩像，身份清晰，画面干净，服饰和时代质感服从本任务"""
```

TOML 任务类型等价于：

```ts
type ImageGenerationTask = {
  id: string
  target_image_path: string   // 若图片暂时不存在，行尾注释 # todo；若已存在且本轮未改提示词，行尾注释 # done；若本轮修改了该任务提示词或参考图，即使图片文件已存在，也必须改为 # todo
  reference_image_paths: string[]
  aspect_ratio: string         // 建议画幅，如 "3:4"、"2:3"、"1:1"、"16:9"、"9:16"；必须服务图片类型与主体形状
  prompt: string              // [基础美术]+[具体内容]+[生成要求]；禁止使用 negative prompt
}

type ImageGenerationPlan = ImageGenerationTask[]
```

### TOML 路径与目录约定

当前项目默认图片资产目录：

```text
资产/
  人物/
    角色名字/
      角色名_源头脸部特写_3x4.png
      角色名_立绘_.png
  道具/
    道具名.png
  EP1/
    场景/
      场景名.png
    概念剧照/
      名.png
```

路径写法要求：

- `target_image_path` 和 `reference_image_paths` 使用相对路径，不使用绝对路径。
- `aspect_ratio` 是必填字段；角色脸部特写通常用 `3:4`；单人单面全身立绘可用 `2:3`；正反同框立绘必须用 `1:1`；细长竖向道具优先 `9:16` 或 `2:3`；环境与概念剧照常用 `16:9`，具体以主体形状和后续用途为准。
- 角色统一放入 `资产/人物/<角色名字>/`；道具统一放入 `资产/道具/`；分集场景与概念剧照放入对应 `资产/EPxx/场景/`、`资产/EPxx/概念剧照/`。
- 行尾状态注释是计划执行状态的一部分：`# done` 表示现有图片可以继续沿用；`# todo` 表示需要重新生成或首次生成。
- 只要本轮修改了某个任务的 `prompt`、`reference_image_paths`、身份年龄或关键约束，该任务必须标为 `# todo`，不能因为目标图片文件已经存在而保留 `# done`。

### TOML 人物图硬规则

脸部大头照 / 源头脸 / 年龄状态脸的详细写法已经拆为子技能：`references/headshot-generation.md`。凡写或修订 `CH-*FACE`、`*_源头大头照_3x4.png`、`*_源头脸部特写_3x4.png`、`*_状态脸_3x4.png` 任务，必须先读取该子技能。

批量人物反同脸规则已经拆为子技能：`references/character-appearance-differentiation.md`。凡同一文件内有 2 个以上人物脸部任务，或用户指出“同脸 / 长得太像”，必须先读取该子技能，建立角色外貌差异锚点后再写 FACE/STAND prompt。

- 扫描所有 `CH-*` 人物图任务，不只检查主角；所有人物脸部特写、立绘、状态图都必须写明确视觉年龄或年龄段。
- 大头照 / 源头脸只定义脸部身份：画面从头顶到锁骨下方少量肩部，柔和虚化的低细节历史环境背景，人物脸部清晰突出，不承担完整服装、显眼道具、清晰剧情场景或氛围剧照功能。
- 同一人物存在多个年龄段时，源头脸优先选择青年时代或更年轻的非儿童阶段；儿童阶段最后做；中年、老年只作为由源头脸派生的年龄状态图。
- 视觉年龄和长相描述必须依据剧本、镜头组、制作备注或原典线索判断；不能只从角色名机械推断。
- 每个角色源头脸必须作为独立身份生成，不能因为上下文或相邻任务而长得像其他角色；如果用户指出某张脸像另一个角色，必须把该脸部任务标为 `# todo`，并在 prompt 中改写该角色自己的正向身份特征、年龄、脸型、眉眼、气质和背景要求。
- 正反同框方图立绘的详细写法已经拆为子技能：`references/front-back-standee-generation.md`。凡写或修订 `CH-*STAND`、`*_立绘_.png`、`*_立绘_1x1.png`、`*_正背立绘_1x1.png`，或 prompt 中出现“正反同框 / 正面与背面左右并列 / 1:1方图”的人物资产任务，必须先读取该子技能。
- 立绘必须是资产参考图，不是氛围剧照：背景必须写“纯白背景 / white background”，不得使用灰色影棚、纹理墙、暗背景、场景背景或电影光效背景。
- 正反同框立绘必须写清“正面与背面同框、左右并列、1:1 方图、纯白背景、全身完整不裁切、两个人物比例一致”；对应 `aspect_ratio = "1:1"`。
- 动物、群体、幻影、塑像等非标准人物任务也要写等价年龄/状态约束，例如 `成年战马`、`老年城中人群`、`青年至中年论师群像`、`佛像不按真人年龄处理而按古代造像面相`。
- 批量修订人物图时，凡新增或改写年龄约束的任务，一律标为 `# todo`。

## 编号规则

推荐使用稳定编号：

- `CH-01`：角色 / 状态参考图。
- `ENV-01`：环境 / 场景空镜。
- `PROP-01`：道具 / 服装 / 物件。
- `STILL-01`：由镜头组导出的剧情概念剧照编号；不得使用表示最终成片帧的命名。
- `PAGE-01`：漫画页 / 分格页。

编号一旦写入文档，后续尽量不要重排；取消时保留条目并标注“取消”。

## 长篇项目角色与道具资产目录规则

跨集复用的角色资产应从单集目录抽出，集中放在项目级 `人物/` 目录。目录必须浅，不要为同一角色再拆 `源头大头照/`、`立绘/`、`服装道具/` 等多层；用统一文件名区分类型：

```text
人物/
  角色名/
    角色名_源头大头照.png
    角色名_25岁正式脸.png
    角色名_EP01长安法师袍服装图.png
    角色名_EP01长安法师袍立绘.png
    角色名_EP19重伤状态.png
    角色名_源头大头照_不采用.png
```

规则：

- 源头大头照定义“这个人是谁”，必须是近距离身份头像；使用柔和虚化的低细节历史环境背景，画面从头顶到锁骨下方少量肩部，允许少量符合身份的衣领 / 上衣质感，但不能带复杂服装、显眼道具或清晰剧情环境。
- 年龄阶段脸只在剧情跨多年且面部成熟度确实变化时新增；新增时必须只参考源头脸，不能参考上一年龄脸、立绘或概念剧照。
- 服装图应先于对应人物立绘生成；立绘必须同时引用“正式脸 + 服装图”。
- 任何带有人物脸部的照片都必须且仅能用源头脸锁身份；服装图、立绘、概念剧照不能成为脸部身份来源。
- 文件名含 `不采用` 或 `弃用` 的图片为废弃/归档素材，任何后续图片都不得以它作为参考图。
- 单集 `资产/EPxx/` 下保留场景、道具、概念剧照等本集生产资产；单集图片计划通过相对路径引用项目级 `人物/`。

跨集复用的重要道具应从单集目录抽出，集中放在项目级 `道具/` 目录。剧情上同一个道具必须只对应一个主参考图，不能因为进入新集数就重新生成一张外观不同的同名道具：

```text
道具/
  锡杖.png
  行囊.png
  经卷.png
  念珠.png
  锡杖_损坏状态_EP12.png
  行囊_打开状态_EP04.png
  经卷_展开状态_EP03.png
```

规则：

- **同一剧情道具只生成一次主图**：如果 EP01 已有 `行囊.png`，EP02、EP03 等后续计划必须引用同一张图，不得再写 `资产/EP02/道具/P02_行囊.png` 这类重复生成项。
- **状态图不是新道具**：打开、损坏、沾血、包裹展开、经卷摊开等是同一物件的状态图，文件名必须标注状态和首次使用集数；后续相同状态也复用。
- **只有剧情上确实不是同一个物件才新增**：例如不同角色各自有不同包袱、不同经卷、不同法器，必须在名称中标明归属或区别，如 `玄奘_行囊.png`、`商队_货袋.png`。
- **计划前先查重**：写每集道具计划前，必须检查项目级 `道具/`、前集 `资产/EPxx/道具/` 和已有图片生成计划；发现同一剧情道具已存在时，条目写为“可复用参考图”，不列入“待生成”。
- **节省生成算力是硬约束**：重复生成同一道具会造成逻辑穿帮和模型时间浪费；除非有明确剧情理由，不得为每集复制一套锡杖、行囊、经卷等常用道具。

## 提示词重点

### 角色图

- 身份参考图：只在角色第一次出现或进入新年龄阶段时生成。
- 服装 / 状态参考图：跨集角色换装时先生成服装图，再用“正式脸 + 服装图”生成立绘。
- 完整全身、半身、头肩；立绘背景必须是纯白背景，不能只写“干净背景 / 简化背景”。
- 正反同框立绘必须使用 `1:1` 方图，正面与背面左右并列，全身完整不裁切，比例一致。
- 姿态、表情、状态。
- 避免换脸、重复人物、错误服装、过度复杂配件。

### 环境图

- 无人物或无命名角色；如果需要群演，必须写明时代、身份、数量、距离和不可抢主体。
- 地点、时间、空间结构。
- 后续动作需要的站位、路线、入口、遮挡、道具位置。
- 室内空间要可信：家具、经案、蒲团、书架、灯烛、柱网等必须留出真实通行距离和视觉呼吸感，避免成列陈设过密、过满、像仓库或展厅。
- 历史场景的柱子、梁架、帷幔、屏风等结构件要“够用但不滥用”；不要让模型自动生成过多柱子或重复柱阵，除非源文档明确要求密集柱廊。
- 古代/历史空镜必须排除现代污染：远处人物也不能出现现代发型、现代服装、现代鞋包、手机、口罩、相机、游客姿态、现代灯具、电线、标牌、玻璃幕墙、现代路面和现代陈设。
- 避免空间不连贯、主体误入、文字/Logo/水印、细节过满。

### 道具图

- 先判断是否为跨集复用的重要道具；如果是同一个剧情道具，只引用项目级 `道具/` 中已确认主图，不要在每集重复生成。
- 道具数量和摆放方式。
- 尺度、材质、结构可读性。
- 道具画幅必须服务物体形状：锡杖、长矛、法杖、旗杆、卷轴立轴、挂轴、长条包裹等细长竖向物体，优先用竖图 `9:16` 或 `2:3`，不要用正方形；这样能把更多像素用于物体本身，避免主体过小、留白过多或细节糊掉。
- 只有接近等宽等高的道具、组合道具平铺、图标式物件，才考虑 `1:1` 方图；横向长物体可用横图 `16:9` 或 `3:2`。
- 是否需要展示打开、闭合、分解或组合状态。
- 状态变化要标注为同一道具的状态图，而不是新道具；只有剧情明确是另一个物件时才新增。
- 避免品牌文字、玩具感、结构不可读。

### 概念剧照

- 谁在场。
- 在哪里。
- 正在做什么。
- 危险或冲突从哪里来。
- 对应源文档哪一段 / 哪个镜头组。
- 为什么这一组只需要一张概念剧照，或为什么必须拆成多张。
- 人数和角色不要自动增减。

## 取消项写法（TOML only）

如果某张图不再生成，不要删除任务。保留对应 `[[tasks]]`，将 `target_image_path` 行尾标为 `# cancel`，并用 TOML comments 写明原因和替代处理。不要使用 TOML 小标题或列表。

```toml
# cancel_reason: 该角色只作为伏笔存在，当前章节不提前给出明确形象。
# cancel_handling: 后续概念剧照只用环境、声音、角色反应或不可辨认剪影暗示。
[[tasks]]
id = "CH-05"
target_image_path = "资产/人物/某角色/某角色_源头脸部特写_3x4.png" # cancel
reference_image_paths = []
aspect_ratio = "3:4"
prompt = """【基础美术】...
【具体内容】...
【生成要求】..."""
```

## 自检规则（不要输出为 TOML；需要写入计划时改成 TOML comments）

- 是否已有镜头组 / 分镜节拍 / shot table；如果没有，是否已停止并提醒用户先做镜头组。
- 如果是多集 / 数百任务 / 大型 TOML，是否已保持 `图片生成计划/` 一层批次文件结构：`1.toml`、`2.toml`、`3.toml`……，没有 `00_制作顺序/`、`01_主人物/`、`02_配角/`、`03_环境/`、`04_服装道具/`、`05_分集/` 等计划子目录。
- 是否确认文件名数字顺序就是总执行顺序，单个 TOML 内 `[[tasks]]` 的自然顺序就是本批次生成顺序，不再额外维护制作顺序索引。
- 是否只依据当前镜头组及其对应源文档，没有引入旧剧情或废弃设定。
- 是否每张图都有保存路径、类型、建议画幅、参考图和提示词。
- 是否所有路径都是相对路径，能从本 TOML 文件位置正确跳转。
- 是否明确哪些图是“待生成”，哪些图是“可复用”。
- 是否先检查项目级 `道具/`、前集道具和旧图片计划，避免为同一剧情道具跨集重复生成。
- 跨集重要道具是否集中放在项目级 `道具/` 目录，并由每集图片计划引用，而不是复制到每集 `资产/EPxx/道具/`。
- 道具状态图是否标明“打开 / 损坏 / 展开 / 特殊状态”，而不是误当成新道具。
- 道具画幅是否匹配形状：细长竖向道具是否使用 `9:16` 或 `2:3`，而不是默认方图。
- 每个计划文件内是否严格按 ENV/environment → objects/tools/props/clothes/costumes → CH/characters/roles → STILL/conceptual scene shots 排序。
- 是否按镜头组导出概念剧照，而不是为每个单镜头机械生成。
- 多地点集的每张概念剧照是否选择了一个明确可生成地点；不要写 `画面地点为“地点A、地点B、地点C”` 让模型混合空间，必要时拆成 `STILL-xxA / STILL-xxB`。
- 纯环境 / 场景空镜的视觉锚点是否已去人物化；不要把父子对坐、送行队伍、人海、王使入寺、众王礼拜等人物/动作锚点直接写进空镜，除非条目改为事件/群像建立镜头。
- 多地点集 ENV 是否不仅“本图只锁定某地点”，还检查视觉锚点是否属于该地点，避免把其他城市、转场或回忆空间的道具/行动塞进同一空镜。
- 每个命名角色、年龄阶段和幻影角色是否已有项目级身份锚点；没有锚点时，先补源头脸/服装图，或明确写成背影、剪影、远景、群演、不可识别。
- 是否每张参考图都说明了具体参考用途。
- 是否提示词包含 `【基础美术风格】`、`【具体生成内容】` 和必要的 `【生成要求】`。
- 是否人物数量、命名角色数量、重复人物规则写清楚。
- 场景空镜是否避免了室内成列陈设过密、柱子过多、空间像仓库/展厅等不真实问题。
- 历史/古代场景是否检查并禁止了远处现代人、现代服饰发型、手机相机、现代灯具、电线、标牌、玻璃幕墙等时代污染。
- 是否没有让图片计划反向改写故事、剧本或镜头组。
- 是否没有提交图片生成任务；本 TOML 文件只是计划。
