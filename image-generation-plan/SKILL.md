# 图片生成计划

## 适用场景

当需要把故事、剧本、镜头组、制作包、漫画章节或已有资产，整理成一份“后续逐张生成图片”的计划文档时，使用本技能。

本技能只规定计划文档的格式、写法、信息完整度和约束纪律，不绑定任何具体项目画风。

## 核心原则

1. **只写计划，不生成图片**
   - 文档只包含生成顺序、保存路径、参考图、提示词和检查项。
   - 不调用图片生成工具，不提交任务，不把“待生成”误写成“已生成”。

2. **计划服从源文档**
   - 图片计划必须服务已有故事、剧本、镜头组或制作包。
   - 不借图片计划新增剧情、改写人物关系、调换镜头顺序或补不存在的设定。
   - 源文档冲突时，先标注冲突或等待确认，不在图片计划里自行裁决。

3. **镜头组是关键帧来源，不是单镜头表**
   - 每张剧情关键帧优先对应一个镜头组，必要时多个镜头组合并为一张关键帧。
   - 不默认为组内每个节拍生成一张图；只有身份、空间、道具或动作结果明显不同时才拆成多张。
   - 漫画页和视频关键帧也按镜头组组织，避免把长故事直接变成零散单图。

4. **角色和道具中心管理，先复用后新增**
   - 长篇 / 多集项目中，跨集角色的脸、年龄段、服装状态和立绘属于项目级角色资产，放在 `人物/<角色名>/` 下，不跟随单集目录反复生成；同一角色的图片直接放在同一个角色文件夹内，不再按 `源头大头照/`、`立绘/` 等拆多层。
   - 长篇 / 多集项目中，跨集使用的重要道具属于项目级道具资产，放在项目级 `道具/` 目录；如果剧情上是同一个道具（如锡杖、行囊、经卷、念珠、书箱、马鞍等），只能生成一张主参考图，后续各集只引用它，不要在每集 `资产/EPxx/道具/` 下重复生成同名道具。
   - 单集目录主要保存本集场景、本集一次性道具、概念剧照、镜头组和图片生成计划；单集计划引用项目级人物与项目级道具资产。
   - 写计划前必须先检查已有 `人物/`、`道具/`、前集资产索引和已生成图片；能复用的角色/道具先复用，只有剧情明确出现新道具、道具外观永久变化、损坏/打开/特殊状态需要单独表现时，才新增道具图。
   - 推荐生成顺序：环境 / 场景 → 人物源头正脸 → 项目级常用道具 → 服装图 → 人物立绘（正脸 + 服装图）→ 本集一次性道具 → 概念剧照。
   - 后面的图片需要引用前面的图片时，先把前序图片写入计划并给出保存路径。

5. **跨集角色只定一次源头脸**
   - 角色第一次出现时生成并确定唯一源头大头照 / 正脸图，作为项目级身份锚点。源头脸必须类似证件照：干净浅色背景、正面或接近正面、无复杂服装、无剧情环境、无电影剧照光效，只定义身份，不定义服装和场景。
   - 后续集数不再为同一年龄阶段重复生成该角色大头照；只在服装、装备、剧情状态变化时补服装图和人物立绘。
   - 如果剧情跨越多年，可按关键年龄阶段生成新的年龄脸；所有年龄脸以及所有带有人物脸部的图片，都必须且只能参考源头脸来锁身份，不能参考上一阶段脸、剧照或立绘来传递身份。

6. **路径、参考图、提示词必须可执行**
   - 每张图都有明确保存路径。
   - 参考图必须写明来源和用途。
   - 提示词必须能直接交给图片生成工具，而不是只写剧情摘要。

7. **约束写在计划里，不靠记忆执行**
   - 人物身份、人数、禁用素材、尺度边界、参考图用途、避免项都必须写清楚。

## 推荐文档结构

```markdown
# 《项目 / 集数 / 章节》图片生成计划

> 目标：为 [源文档](相对路径) 整理后续逐张生成图片的计划。
> 本文档只写生成计划、保存路径、提示词、参考图；**不生成图片**。
> 本计划只服务当前源文档；**不修改故事、不修改镜头组/制作包**。

## 0. 本次图片生成总规则

### 0.1 源文档与范围
### 0.2 固定参考资产 / 美术规则
### 0.3 角色、人数与连续性规则
### 0.4 尺度、禁用内容与特殊注意事项

## 1. 生成顺序总览

## 2. 角色 / 状态参考图

## 3. 场景 / 环境图

## 4. 道具 / 服装 / 物件参考图

## 5. 剧情关键帧 / 漫画页 / 视频关键帧

## 6. 生成前检查清单
```

章节可按项目需要增删，但应保持“总规则 → 顺序总览 → 分类型图片计划 → 检查清单”的顺序。

## 总规则写法

### 源文档与范围

必须说明：

- 计划依据哪些文件：故事、剧本、镜头组、制作包、角色设定、资产索引等。
- 本计划覆盖的集数、章节、场景范围。
- 哪些旧设定、废弃目录、未采用素材、过时剧情不能使用。
- 图片计划不得反向改写剧情或镜头组。

### 固定参考资产 / 美术规则

必须说明：

- 固定参考图有哪些。
- 每张参考图用于什么：身份、服装、场景、光感、构图、材质、情绪等。
- 哪些内容只参考风格，不能复制具体物件或角色。
- 后续提示词采用什么统一公式。

通用提示词公式：

```text
【基础美术风格】<只写画面语言、风格、材质、光影、细节密度等通用规则>
【具体生成内容】<写主体、动作、地点、道具、状态、人数、构图目的>
【避免】<写本图最关键的禁项；不要堆砌无关负面词>
```

注意：

- `【基础美术风格】` 不应混入具体人物、剧情、道具或项目专属设定。
- `【具体生成内容】` 承担主体、动作、地点和剧情信息。
- `【避免】` 只写会显著影响生成质量或连续性的禁项。

### 角色、人数与连续性规则

必须说明：

- 每个主要角色的身份锚点参考图。
- 哪些图必须保持同一张脸、同一服装、同一道具或同一场景结构。
- 同一画面内能出现几个命名角色。
- 是否禁止同一人物重复出现、残影式连续动作、自动加人。
- 群演、路人、士兵、怪物等是否有精确数量要求。

## 单张图片条目格式

```markdown
### <编号> <图片名称>

- **保存路径**：[待生成：文件名.png](相对/保存路径/文件名.png)
- **类型**：<角色状态参考 / 纯环境 / 道具参考 / 概念剧照 / 漫画页 / 视频关键帧>
- **建议画幅**：<横图 16:9 / 竖图 2:3 / 横图 4:3 / 方图 1:1 / 按项目需要>
- **参考图**：
  - 图1：[参考图文件名](相对路径) — <只写该参考图用于什么>
  - 图2：[参考图文件名](相对路径) — <只写该参考图用于什么>
- **提示词**：

```text
【基础美术风格】……
【具体生成内容】……
【避免】……
```
```

## 编号规则

推荐使用稳定编号：

- `CH-01`：角色 / 状态参考图。
- `ENV-01`：环境 / 场景空镜。
- `PROP-01`：道具 / 服装 / 物件。
- `KF-01` 或 `STILL-01`：由镜头组导出的剧情关键帧 / 概念剧照。
- `PAGE-01`：漫画页 / 分格页。

编号一旦写入文档，后续尽量不要重排；取消时保留条目并标注“取消”。

## 长篇项目角色与道具资产目录规则

跨集复用的角色资产应从单集目录抽出，集中放在项目级 `人物/` 目录。目录必须浅，不要为同一角色再拆 `源头大头照/`、`立绘/`、`服装道具/` 等多层；用统一文件名区分类型：

```text
人物/
  角色名/
    角色名_源头大头照.png
    角色名_25岁正式脸.png
    角色名_EP01长安法师袍服装图.png
    角色名_EP01长安法师袍立绘.png
    角色名_EP19重伤状态.png
    角色名_源头大头照_不采用.png
```

规则：

- 源头大头照定义“这个人是谁”，必须是干净浅色背景的证件照式正脸，不是剧照，也不能带复杂服装或剧情环境。
- 年龄阶段脸只在剧情跨多年且面部成熟度确实变化时新增；新增时必须只参考源头脸，不能参考上一年龄脸、立绘或概念剧照。
- 服装图应先于对应人物立绘生成；立绘必须同时引用“正式脸 + 服装图”。
- 任何带有人物脸部的照片都必须且仅能用源头脸锁身份；服装图、立绘、概念剧照不能成为脸部身份来源。
- 文件名含 `不采用` 或 `弃用` 的图片为废弃/归档素材，任何后续图片都不得以它作为参考图。
- 单集 `资产/EPxx/` 下保留场景、道具、概念剧照等本集生产资产；单集图片计划通过相对路径引用项目级 `人物/`。

跨集复用的重要道具应从单集目录抽出，集中放在项目级 `道具/` 目录。剧情上同一个道具必须只对应一个主参考图，不能因为进入新集数就重新生成一张外观不同的同名道具：

```text
道具/
  锡杖.png
  行囊.png
  经卷.png
  念珠.png
  锡杖_损坏状态_EP12.png
  行囊_打开状态_EP04.png
  经卷_展开状态_EP03.png
```

规则：

- **同一剧情道具只生成一次主图**：如果 EP01 已有 `行囊.png`，EP02、EP03 等后续计划必须引用同一张图，不得再写 `资产/EP02/道具/P02_行囊.png` 这类重复生成项。
- **状态图不是新道具**：打开、损坏、沾血、包裹展开、经卷摊开等是同一物件的状态图，文件名必须标注状态和首次使用集数；后续相同状态也复用。
- **只有剧情上确实不是同一个物件才新增**：例如不同角色各自有不同包袱、不同经卷、不同法器，必须在名称中标明归属或区别，如 `玄奘_行囊.png`、`商队_货袋.png`。
- **计划前先查重**：写每集道具计划前，必须检查项目级 `道具/`、前集 `资产/EPxx/道具/` 和已有图片生成计划；发现同一剧情道具已存在时，条目写为“可复用参考图”，不列入“待生成”。
- **节省生成算力是硬约束**：重复生成同一道具会造成逻辑穿帮和模型时间浪费；除非有明确剧情理由，不得为每集复制一套锡杖、行囊、经卷等常用道具。

## 提示词重点

### 角色图

- 身份参考图：只在角色第一次出现或进入新年龄阶段时生成。
- 服装 / 状态参考图：跨集角色换装时先生成服装图，再用“正式脸 + 服装图”生成立绘。
- 完整全身、半身、头肩、干净背景或简化背景。
- 姿态、表情、状态。
- 避免换脸、重复人物、错误服装、过度复杂配件。

### 环境图

- 无人物或无命名角色；如果需要群演，必须写明时代、身份、数量、距离和不可抢主体。
- 地点、时间、空间结构。
- 后续动作需要的站位、路线、入口、遮挡、道具位置。
- 室内空间要可信：家具、经案、蒲团、书架、灯烛、柱网等必须留出真实通行距离和视觉呼吸感，避免成列陈设过密、过满、像仓库或展厅。
- 历史场景的柱子、梁架、帷幔、屏风等结构件要“够用但不滥用”；不要让模型自动生成过多柱子或重复柱阵，除非源文档明确要求密集柱廊。
- 古代/历史空镜必须排除现代污染：远处人物也不能出现现代发型、现代服装、现代鞋包、手机、口罩、相机、游客姿态、现代灯具、电线、标牌、玻璃幕墙、现代路面和现代陈设。
- 避免空间不连贯、主体误入、文字/Logo/水印、细节过满。

### 道具图

- 先判断是否为跨集复用的重要道具；如果是同一个剧情道具，只引用项目级 `道具/` 中已确认主图，不要在每集重复生成。
- 道具数量和摆放方式。
- 尺度、材质、结构可读性。
- 道具画幅必须服务物体形状：锡杖、长矛、法杖、旗杆、卷轴立轴、挂轴、长条包裹等细长竖向物体，优先用竖图 `9:16` 或 `2:3`，不要用正方形；这样能把更多像素用于物体本身，避免主体过小、留白过多或细节糊掉。
- 只有接近等宽等高的道具、组合道具平铺、图标式物件，才考虑 `1:1` 方图；横向长物体可用横图 `16:9` 或 `3:2`。
- 是否需要展示打开、闭合、分解或组合状态。
- 状态变化要标注为同一道具的状态图，而不是新道具；只有剧情明确是另一个物件时才新增。
- 避免品牌文字、玩具感、结构不可读。

### 关键帧

- 谁在场。
- 在哪里。
- 正在做什么。
- 危险或冲突从哪里来。
- 对应源文档哪一段 / 哪个镜头组。
- 为什么这一组只需要一张关键帧，或为什么必须拆成多张。
- 人数和角色不要自动增减。

## 取消项写法

如果某张图不再生成，不要直接删除。保留条目，明确状态、原因和替代处理，避免后续误补。

```markdown
### CH-05 某角色参考图（取消）

- **状态**：取消生成，不再作为角色图或参考图。
- **原因**：该角色只作为伏笔存在，当前章节不提前给出明确形象。
- **处理方式**：后续关键帧只用环境、声音、角色反应或不可辨认剪影暗示；禁止画出完整可识别形象。
```

## 检查清单

- [ ] 是否只依据当前源文档，没有引入旧剧情或废弃设定。
- [ ] 是否每张图都有保存路径、类型、建议画幅、参考图和提示词。
- [ ] 是否所有路径都是相对路径，能从本文档位置正确跳转。
- [ ] 是否明确哪些图是“待生成”，哪些图是“可复用”。
- [ ] 是否先检查项目级 `道具/`、前集道具和旧图片计划，避免为同一剧情道具跨集重复生成。
- [ ] 跨集重要道具是否集中放在项目级 `道具/` 目录，并由每集图片计划引用，而不是复制到每集 `资产/EPxx/道具/`。
- [ ] 道具状态图是否标明“打开 / 损坏 / 展开 / 特殊状态”，而不是误当成新道具。
- [ ] 道具画幅是否匹配形状：细长竖向道具是否使用 `9:16` 或 `2:3`，而不是默认方图。
- [ ] 是否先生成身份锚点、场景、道具，再生成关键帧。
- [ ] 是否按镜头组导出关键帧，而不是为每个单镜头机械生成。
- [ ] 是否每张参考图都说明了具体参考用途。
- [ ] 是否提示词包含 `【基础美术风格】`、`【具体生成内容】` 和必要的 `【避免】`。
- [ ] 是否人物数量、命名角色数量、重复人物规则写清楚。
- [ ] 场景空镜是否避免了室内成列陈设过密、柱子过多、空间像仓库/展厅等不真实问题。
- [ ] 历史/古代场景是否检查并禁止了远处现代人、现代服饰发型、手机相机、现代灯具、电线、标牌、玻璃幕墙等时代污染。
- [ ] 是否没有让图片计划反向改写故事、剧本或镜头组。
- [ ] 是否没有提交图片生成任务；本文档只是计划。
