# 图片生成计划

## 适用场景

当需要把故事、剧本、镜头组、制作包、漫画章节或已有资产，整理成一份“后续逐张生成图片”的计划文档时，使用本技能。

本技能只规定计划文档的格式、写法、信息完整度和约束纪律，不绑定任何具体项目画风。

## 核心原则

1. **只写计划，不生成图片**
   - 文档只包含生成顺序、保存路径、参考图、提示词和检查项。
   - 不调用图片生成工具，不提交任务，不把“待生成”误写成“已生成”。

2. **图片生成计划必须等镜头组之后再做**
   - 图片生成计划不能从剧本直接跳过来生成；必须先有镜头组 / 分镜节拍 / shot table 作为直接依据。
   - 如果用户只提供剧本、故事或分集大纲，却要求直接制作图片生成计划，必须先打断并提醒：需要先完成镜头组；不能为了省步骤直接写图片计划。
   - 只有当镜头组已经存在，或用户明确提供等价的镜头组文档时，才继续制作图片生成计划。

3. **计划服从源文档**
   - 图片计划必须服务已有镜头组，并回溯对应故事、剧本或制作包。
   - 不借图片计划新增剧情、改写人物关系、调换镜头顺序或补不存在的设定。
   - 源文档冲突时，先标注冲突或等待确认，不在图片计划里自行裁决。

4. **镜头组是概念剧照来源，不是单镜头表**
   - 每张剧情概念剧照优先对应一个镜头组，必要时多个镜头组合并为一张概念剧照。概念剧照不是最终成片中 exactly 出现的 frame，而是服务人物、场景、情绪、动作关系和视频生成参考的宽用途剧照。
   - 不默认为组内每个节拍生成一张图；只有身份、空间、道具或动作结果明显不同时才拆成多张。
   - 漫画页和视频概念剧照也按镜头组组织，避免把长故事直接变成零散单图。

4. **角色和道具中心管理，先复用后新增**
   - 长篇 / 多集项目中，跨集角色的脸、年龄段、服装状态和立绘属于项目级角色资产，放在 `人物/<角色名>/` 下，不跟随单集目录反复生成；同一角色的图片直接放在同一个角色文件夹内，不再按 `源头大头照/`、`立绘/` 等拆多层。
   - 长篇 / 多集项目中，跨集使用的重要道具属于项目级道具资产，放在项目级 `道具/` 目录；如果剧情上是同一个道具（如锡杖、行囊、经卷、念珠、书箱、马鞍等），只能生成一张主参考图，后续各集只引用它，不要在每集 `资产/EPxx/道具/` 下重复生成同名道具。
   - 单集目录主要保存本集场景、本集一次性道具、概念剧照、镜头组和图片生成计划；单集计划引用项目级人物与项目级道具资产。
   - 写计划前必须先检查已有 `人物/`、`道具/`、前集资产索引和已生成图片；能复用的角色/道具先复用，只有剧情明确出现新道具、道具外观永久变化、损坏/打开/特殊状态需要单独表现时，才新增道具图。
   - 推荐生成顺序：环境 / 场景 → 人物源头正脸 → 项目级常用道具 → 服装图 → 人物立绘（正脸 + 服装图）→ 本集一次性道具 → 概念剧照。
   - 后面的图片需要引用前面的图片时，先把前序图片写入计划并给出保存路径。

5. **跨集角色只定一次源头脸**
   - 角色第一次出现时生成并确定唯一源头大头照 / 正脸图，作为项目级身份锚点。源头脸必须类似证件照：干净浅色背景、正面或接近正面、无复杂服装、无剧情环境、无电影剧照光效，只定义身份，不定义服装和场景。
   - 后续集数不再为同一年龄阶段重复生成该角色大头照；只在服装、装备、剧情状态变化时补服装图和人物立绘。
   - 如果剧情跨越多年，可按关键年龄阶段生成新的年龄脸；所有年龄脸以及所有带有人物脸部的图片，都必须且只能参考源头脸来锁身份，不能参考上一阶段脸、剧照或立绘来传递身份。

6. **路径、参考图、提示词必须可执行**
   - 每张图都有明确保存路径。
   - 参考图必须写明来源和用途。
   - 提示词必须能直接交给图片生成工具，而不是只写剧情摘要。

7. **约束写在计划里，不靠记忆执行**
   - 人物身份、人数、禁用素材、尺度边界、参考图用途、避免项都必须写清楚。

## 推荐文档结构

```markdown
# 《项目 / 集数 / 章节》图片生成计划

> 目标：为 [源文档](相对路径) 整理后续逐张生成图片的计划。
> 本文档只写生成计划、保存路径、提示词、参考图；**不生成图片**。
> 本计划只服务当前源文档；**不修改故事、不修改镜头组/制作包**。

## 0. 本次图片生成总规则

### 0.1 源文档与范围
### 0.2 固定参考资产 / 美术规则
### 0.3 角色、人数与连续性规则
### 0.4 尺度、禁用内容与特殊注意事项

## 1. 生成顺序总览

## 2. 角色 / 状态参考图

## 3. 场景 / 环境图

## 4. 道具 / 服装 / 物件参考图

## 5. 剧情概念剧照 / 漫画页 / 视频概念剧照

## 6. 生成前检查清单
```

章节可按项目需要增删，但应保持“总规则 → 顺序总览 → 分类型图片计划 → 检查清单”的顺序。

## 总规则写法

### 源文档与范围

必须说明：

- 计划依据哪些镜头组文件；镜头组对应哪些故事、剧本、制作包、角色设定、资产索引等。
- 本计划覆盖的集数、章节、镜头组范围。
- 如果没有镜头组，停止制作图片生成计划，先提醒用户补做镜头组。
- 哪些旧设定、废弃目录、未采用素材、过时剧情不能使用。
- 图片计划不得反向改写剧情或镜头组。

### 固定参考资产 / 美术规则

必须说明：

- 固定参考图有哪些。
- 每张参考图用于什么：身份、服装、场景、光感、构图、材质、情绪等。
- 哪些内容只参考风格，不能复制具体物件或角色。
- 后续提示词采用什么统一公式。

通用提示词公式：

```text
【基础美术风格】<电影项目只写硬摄影规格：摄影机型号、记录格式、镜头品牌/系列/焦段、T 值、画幅比例；不要写文学化口号，也不要写机位、景别、运动方式、光源解释>
【具体生成内容】<写主体、动作、地点、道具、状态、人数、构图目的>
【避免】<写本图最关键的禁项；不要堆砌无关负面词>
```

注意：

- `【基础美术风格】` 不应混入具体人物、剧情、道具或项目专属设定。
- 电影 / 短剧 / 影视项目的 `【基础美术风格】` 应采用摄影部门语言，而不是抽象美术口号。必须明确摄影机和镜头型号，例如 `ARRI ALEXA 35 + Cooke Anamorphic/i 40mm`、`Sony VENICE 2 + Panavision Primo 50mm` 等；具体型号应服从项目统一摄影方案。
- `【基础美术风格】` 只放摄影机、记录格式、镜头系列/焦段、T 值、画幅比例等硬规格；不要写“安全构图、固定机位、光源依据、背景只交代空间、不使用现代柔光箱效果”等调度/说明文字，这些如必须出现，应放入具体生成内容且保持简短。
- 提示词是给没有上下文的图像模型（如 image2）使用的，必须 self-contained：`【提示词】` 内不能写 `对应镜头组 EPxx-SGxx`、`本图只选择`、`如上文`、`本集`、`镜头组限定` 等依赖外部文档的描述；每条提示词必须直接说明地点、人物、动作、道具、画面边界和禁项。
- 禁止把这类句子当作基础风格：`电影感写实历史纪录片风格，唐代与西域佛教题材，克制构图，真实材质，低饱和暖冷对比，自然尘埃与体积光，空间真实可行，细节清晰但不过度堆砌，无文字无水印。` 其中“真实材质、低饱和暖冷对比、自然尘埃与体积光、空间真实可行、细节清晰但不过度堆砌”等属于文学化/说明书化描述，应删掉或转化为具体摄影、灯光、镜头和场面调度信息。
- `【具体生成内容】` 承担主体、动作、地点和剧情信息；必须短句化、可直接投喂，不写制作备注或索引说明。
- `【避免】` 只写会显著影响生成质量或连续性的禁项。

### 角色、人数与连续性规则

必须说明：

- 每个主要角色的身份锚点参考图。
- 哪些图必须保持同一张脸、同一服装、同一道具或同一场景结构。
- 同一画面内能出现几个命名角色。
- 是否禁止同一人物重复出现、残影式连续动作、自动加人。
- 群演、路人、士兵、怪物等是否有精确数量要求。

## TOML 图片生成计划格式

当用户指定使用 TOML 计划时，输出文件使用 `图片生成计划.toml` 或带集数/版本的同类文件名，根结构为任务数组：

```toml
[[tasks]]
id = "CH-少年玄奘-FACE"
target_image_path = "资产/人物/少年玄奘/少年玄奘_源头脸部特写_3x4.png" # todo
reference_image_paths = []
aspect_ratio = "3:4"
prompt = """【基础美术】ARRI ALEXA 35，Open Gate 4.6K，Cooke Anamorphic/i 40mm，T2.8，2.39:1
【具体内容】少年玄奘源头脸部特写，视觉年龄8岁左右的男童，正面头肩像，干净中性背景，五官清晰，神情聪敏、恭敬、认真到近乎执拗，唐代士族儿童衣冠气质，作为后续同一张脸的身份锚点。
【避免】现代妆发、多人同框、文字、水印"""
```

TOML 任务类型等价于：

```ts
type ImageGenerationTask = {
  id: string
  target_image_path: string   // 若图片暂时不存在，行尾注释 # todo；若已存在且本轮未改提示词，行尾注释 # done；若本轮修改了该任务提示词或参考图，即使图片文件已存在，也必须改为 # todo
  reference_image_paths: string[]
  aspect_ratio: string         // 建议画幅，如 "3:4"、"2:3"、"1:1"、"16:9"、"9:16"；必须服务图片类型与主体形状
  prompt: string              // [基础美术]+[具体内容]+[避免]
}

type ImageGenerationPlan = ImageGenerationTask[]
```

### TOML 路径与目录约定

当前项目默认图片资产目录：

```text
资产/
  人物/
    角色名字/
      角色名_源头脸部特写_3x4.png
      角色名_立绘_.png
  道具/
    道具名.png
  EP1/
    场景/
      场景名.png
    概念剧照/
      名.png
```

路径写法要求：

- `target_image_path` 和 `reference_image_paths` 使用相对路径，不使用绝对路径。
- `aspect_ratio` 是必填字段；角色脸部特写通常用 `3:4`，角色立绘常用 `2:3` 或 `1:1`，细长竖向道具优先 `9:16` 或 `2:3`，环境与概念剧照常用 `16:9`，具体以主体形状和后续用途为准。
- 角色统一放入 `资产/人物/<角色名字>/`；道具统一放入 `资产/道具/`；分集场景与概念剧照放入对应 `资产/EPxx/场景/`、`资产/EPxx/概念剧照/`。
- 行尾状态注释是计划执行状态的一部分：`# done` 表示现有图片可以继续沿用；`# todo` 表示需要重新生成或首次生成。
- 只要本轮修改了某个任务的 `prompt`、`reference_image_paths`、身份年龄或关键约束，该任务必须标为 `# todo`，不能因为目标图片文件已经存在而保留 `# done`。

### TOML 人物图硬规则

- 扫描所有 `CH-*` 人物图任务，不只检查主角；所有人物脸部特写、立绘、状态图都必须写明确视觉年龄或年龄段。
- 视觉年龄必须依据剧本、镜头组、制作备注或原典线索判断；不能只从角色名机械推断。
- 脸部特写尤其要写清：`视觉年龄8岁左右的男童`、`视觉年龄十三岁左右少年僧`、`视觉年龄约44岁中年僧人`、`视觉年龄六十岁以上老年法师` 等。
- 动物、群体、幻影、塑像等非标准人物任务也要写等价年龄/状态约束，例如 `成年战马`、`老年城中人群`、`青年至中年论师群像`、`佛像不按真人年龄处理而按古代造像面相`。
- 批量修订人物图时，凡新增或改写年龄约束的任务，一律标为 `# todo`。

## 编号规则

推荐使用稳定编号：

- `CH-01`：角色 / 状态参考图。
- `ENV-01`：环境 / 场景空镜。
- `PROP-01`：道具 / 服装 / 物件。
- `STILL-01`：由镜头组导出的剧情概念剧照编号；不得使用表示最终成片帧的命名。
- `PAGE-01`：漫画页 / 分格页。

编号一旦写入文档，后续尽量不要重排；取消时保留条目并标注“取消”。

## 长篇项目角色与道具资产目录规则

跨集复用的角色资产应从单集目录抽出，集中放在项目级 `人物/` 目录。目录必须浅，不要为同一角色再拆 `源头大头照/`、`立绘/`、`服装道具/` 等多层；用统一文件名区分类型：

```text
人物/
  角色名/
    角色名_源头大头照.png
    角色名_25岁正式脸.png
    角色名_EP01长安法师袍服装图.png
    角色名_EP01长安法师袍立绘.png
    角色名_EP19重伤状态.png
    角色名_源头大头照_不采用.png
```

规则：

- 源头大头照定义“这个人是谁”，必须是干净浅色背景的证件照式正脸，不是剧照，也不能带复杂服装或剧情环境。
- 年龄阶段脸只在剧情跨多年且面部成熟度确实变化时新增；新增时必须只参考源头脸，不能参考上一年龄脸、立绘或概念剧照。
- 服装图应先于对应人物立绘生成；立绘必须同时引用“正式脸 + 服装图”。
- 任何带有人物脸部的照片都必须且仅能用源头脸锁身份；服装图、立绘、概念剧照不能成为脸部身份来源。
- 文件名含 `不采用` 或 `弃用` 的图片为废弃/归档素材，任何后续图片都不得以它作为参考图。
- 单集 `资产/EPxx/` 下保留场景、道具、概念剧照等本集生产资产；单集图片计划通过相对路径引用项目级 `人物/`。

跨集复用的重要道具应从单集目录抽出，集中放在项目级 `道具/` 目录。剧情上同一个道具必须只对应一个主参考图，不能因为进入新集数就重新生成一张外观不同的同名道具：

```text
道具/
  锡杖.png
  行囊.png
  经卷.png
  念珠.png
  锡杖_损坏状态_EP12.png
  行囊_打开状态_EP04.png
  经卷_展开状态_EP03.png
```

规则：

- **同一剧情道具只生成一次主图**：如果 EP01 已有 `行囊.png`，EP02、EP03 等后续计划必须引用同一张图，不得再写 `资产/EP02/道具/P02_行囊.png` 这类重复生成项。
- **状态图不是新道具**：打开、损坏、沾血、包裹展开、经卷摊开等是同一物件的状态图，文件名必须标注状态和首次使用集数；后续相同状态也复用。
- **只有剧情上确实不是同一个物件才新增**：例如不同角色各自有不同包袱、不同经卷、不同法器，必须在名称中标明归属或区别，如 `玄奘_行囊.png`、`商队_货袋.png`。
- **计划前先查重**：写每集道具计划前，必须检查项目级 `道具/`、前集 `资产/EPxx/道具/` 和已有图片生成计划；发现同一剧情道具已存在时，条目写为“可复用参考图”，不列入“待生成”。
- **节省生成算力是硬约束**：重复生成同一道具会造成逻辑穿帮和模型时间浪费；除非有明确剧情理由，不得为每集复制一套锡杖、行囊、经卷等常用道具。

## 提示词重点

### 角色图

- 身份参考图：只在角色第一次出现或进入新年龄阶段时生成。
- 服装 / 状态参考图：跨集角色换装时先生成服装图，再用“正式脸 + 服装图”生成立绘。
- 完整全身、半身、头肩、干净背景或简化背景。
- 姿态、表情、状态。
- 避免换脸、重复人物、错误服装、过度复杂配件。

### 环境图

- 无人物或无命名角色；如果需要群演，必须写明时代、身份、数量、距离和不可抢主体。
- 地点、时间、空间结构。
- 后续动作需要的站位、路线、入口、遮挡、道具位置。
- 室内空间要可信：家具、经案、蒲团、书架、灯烛、柱网等必须留出真实通行距离和视觉呼吸感，避免成列陈设过密、过满、像仓库或展厅。
- 历史场景的柱子、梁架、帷幔、屏风等结构件要“够用但不滥用”；不要让模型自动生成过多柱子或重复柱阵，除非源文档明确要求密集柱廊。
- 古代/历史空镜必须排除现代污染：远处人物也不能出现现代发型、现代服装、现代鞋包、手机、口罩、相机、游客姿态、现代灯具、电线、标牌、玻璃幕墙、现代路面和现代陈设。
- 避免空间不连贯、主体误入、文字/Logo/水印、细节过满。

### 道具图

- 先判断是否为跨集复用的重要道具；如果是同一个剧情道具，只引用项目级 `道具/` 中已确认主图，不要在每集重复生成。
- 道具数量和摆放方式。
- 尺度、材质、结构可读性。
- 道具画幅必须服务物体形状：锡杖、长矛、法杖、旗杆、卷轴立轴、挂轴、长条包裹等细长竖向物体，优先用竖图 `9:16` 或 `2:3`，不要用正方形；这样能把更多像素用于物体本身，避免主体过小、留白过多或细节糊掉。
- 只有接近等宽等高的道具、组合道具平铺、图标式物件，才考虑 `1:1` 方图；横向长物体可用横图 `16:9` 或 `3:2`。
- 是否需要展示打开、闭合、分解或组合状态。
- 状态变化要标注为同一道具的状态图，而不是新道具；只有剧情明确是另一个物件时才新增。
- 避免品牌文字、玩具感、结构不可读。

### 概念剧照

- 谁在场。
- 在哪里。
- 正在做什么。
- 危险或冲突从哪里来。
- 对应源文档哪一段 / 哪个镜头组。
- 为什么这一组只需要一张概念剧照，或为什么必须拆成多张。
- 人数和角色不要自动增减。

## 取消项写法

如果某张图不再生成，不要直接删除。保留条目，明确状态、原因和替代处理，避免后续误补。

```markdown
### CH-05 某角色参考图（取消）

- **状态**：取消生成，不再作为角色图或参考图。
- **原因**：该角色只作为伏笔存在，当前章节不提前给出明确形象。
- **处理方式**：后续概念剧照只用环境、声音、角色反应或不可辨认剪影暗示；禁止画出完整可识别形象。
```
