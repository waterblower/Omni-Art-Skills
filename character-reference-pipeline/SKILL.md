# 角色参考图入库流程

## 目标

把一次性角色图、群像、临时设定或文字描述整理成可复用的角色资产。核心目标：

- 不抠图、不局部修失败图。
- 不生成景别混杂的角色卡。
- 按角色重要性生成统一风格、无剧情背景的独立参考图。
- 为后续关键帧、漫画页、视频提示词提供稳定身份锚点。

## 输入判断

1. 实际读入现有角色图或群像，确认目标角色数量、位置、外观、服装、武器和关键配饰。
2. 如果用户要求排除某些角色，先列出排除对象和剩余对象。
3. 已有角色名时优先沿用；没有角色名时，根据外观、职能、物种或剧情作用生成短而稳定的名字。
4. 判断角色重要性：长期主角、重要配角、短期配角、三等角色/群演。
5. 如果角色身份或对应关系不清楚，先标注冲突或询问用户，不擅自合并。

## 生成顺序

按角色重要性决定资产数量：

1. **核心主角 / 长期主角**
   - 头部正面
   - 头部背面 / 后脑勺
   - 立绘正面
   - 立绘背面

2. **重要配角 / 短期配角**
   - 头部正面
   - 立绘正面
   - 必要时补立绘背面或状态图

3. **三等角色 / 群演**
   - 立绘正面即可

4. **会换装的重要角色**
   - 先有标准脸部参考和立绘
   - 再做单独服装图
   - 最后做融合服装与身份的换装立绘/状态图

## 通用生图规则

- 角色参考图必须是独立文件，不把全身、半身、大头照和剧情照拼在一张图里。
- 所有角色参考图不得有剧情背景、复杂环境、窗户、街景、灯牌、海报、文字或装饰排版。
- 背景只能使用纯白、纯灰、透明或极简无纹理底色。
- 全身立绘必须完整从头到脚可见，不裁头、不裁脚、不被道具遮挡主体轮廓。
- 正面立绘要自然站立，姿态可复用；不要像动作关键帧。
- 背面立绘必须保持同一发型、服装、鞋履、装备、体型和材质，不得擅自换装。
- 角色长相不要脸谱化；保留可记忆的自然差异、轮廓、眼型、鼻型、嘴型或物种特征。

## 头部正面规则

头部正面 / 大头照 / 脸部参考必须：

- 正脸或近似正脸，直视镜头或清楚展示五官。
- 画面只包含头发、额头、眼睛、鼻子、嘴、脸颊、下巴和必要耳部信息。
- 不出现肩膀、锁骨、胸部、躯干、手、服装卖点或半身构图。
- 无背景、无文字、无海报设计。
- 若生成结果露出肩膀或胸部，裁切或重做后才能作为标准脸部参考。

## 头部背面 / 后脑勺规则

后脑勺图必须：

- 真正从后方展示头部。
- 展示发型后侧、发旋、发尾、耳后轮廓、头饰背面或非人角色颅形结构。
- 不得是侧脸、侧后、回头、半身背影或服装展示。
- 不出现肩背、身体、手或剧情环境。

## 服装与状态参考

只有重要且会换装、受伤、伪装或跨场景变化的角色，才需要单独服装/状态图。

- 单独服装图不作为人物身份参考。
- 服装图应去人物身份化：无可识别脸、头部、表情或演员身份。
- 可使用无头人体模特、隐形模特、衣架、平铺陈列或服装档案板。
- 重点记录服装轮廓、鞋履、配饰、材质、破损、污渍、湿度、装备挂载等。
- 换装图必须同时引用标准脸部参考和服装图：脸来自脸部参考，衣服来自服装图。

## 文件结构建议

```text
角色名/
  角色名-头部正面.png
  角色名-头部背面.png
  角色名-立绘正面.png
  角色名-立绘背面.png
  角色名-服装参考.png        # 可选
  角色名-状态参考.png        # 可选
  角色名.md
```

`角色名.md` 建议包含：

```markdown
# 角色名

一句到两句介绍角色身份、职能和气质。

## 角色图片

![角色名-头部正面](角色名-头部正面.png)
![角色名-头部背面](角色名-头部背面.png)
![角色名-立绘正面](角色名-立绘正面.png)
![角色名-立绘背面](角色名-立绘背面.png)

## 视觉要点

- 后续必须保留的脸部、体型、发型、服装、材质或物种特征。
- 容易漂移或必须避免的点。
```

## 质量检查

- [ ] 全身立绘完整，不裁头、不裁脚、没有多余角色。
- [ ] 图片像重新生成的角色资产，不像抠图、拼贴或截图。
- [ ] 没有景别混杂的角色卡。
- [ ] 没有剧情背景或环境背景。
- [ ] 头部正面是纯脸部特写，没有肩膀、胸部、躯干、手或服装卖点。
- [ ] 后脑勺是真后脑勺，不是侧后或半身背影。
- [ ] 正面和背面是同一角色、同一发型、同一服装、同一鞋履、同一装备和同一体型。
- [ ] 关键识别特征被保留。
- [ ] 不合格图不会进入后续关键帧、漫画页或视频参考链路。
