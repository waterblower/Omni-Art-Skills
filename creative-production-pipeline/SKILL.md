# 通用创作制片流水线

## 适用范围

当任务需要从故事梗概、分集大纲、口播稿、漫画方向、广告创意或零散设定出发，整理成可拍、可画、可生成、可交给后续制作执行的内容时，使用本技能。

它负责通用流程和质量门禁，不绑定任何具体项目、角色、时代、地点、画风或文件路径。如果项目另有专用设定，专用设定只负责具体连续性，本技能负责方法论。

## 总原则

1. **先让故事成立，再让画面成立。** 没有剧本依据的资产，容易漂亮但错误。
2. **先整理源文档，再生产资产。** 不要从零散描述直接跳到关键帧或视频提示词。
3. **一个输出服务一个阶段。** 剧本解决叙事，镜头组解决生成单位和段落节奏，图片计划解决资产顺序，提示词解决生成执行。
4. **剧本不是模板说明。** 剧本只写清“什么人在什么地方做了什么、说了什么”；不要系列、章节、叙事原则、素材依据、本集一句话、旁白基调、制作备注，也不要固定套用开场钩子 / 人物进入 / 核心冲突 / 精神落点 / 结尾钩子。
5. **资产生成服务连续性。** 已失败、身份错误、空间错误或风格错误的图，不得进入后续参考链路。
6. **只在用户明确要求时提交生成任务。** 默认先写剧本、镜头组、计划和提示词，不擅自调用图片/视频生成工具。
7. **镜头组优先于单一分镜。** 面向 AI 图片、视频和漫画时，先把故事整理成可生成的镜头组；单一镜头只作为组内节拍，不作为默认生产单位。
8. **补充细节不能改写核心设定。** 可以补足镜头、动作、空间、道具和制作说明；不能擅自改变人物关系、剧情顺序和核心目标。

## 阶段门禁

按以下顺序推进，不要跳步：

1. **源文档确认**
   - 读取或整理故事、设定、分集大纲、口播稿、广告文案或漫画方向。
   - 明确当前任务的权威来源、废弃来源和未确认信息。
   - 完成标准：能用 3-5 条列出“必须保留 / 不可擅改 / 需要确认”的内容。

2. **剧本或故事稿**
   - 写清场次、地点、角色行动、对白/旁白和必要的剧情转折。
   - 用最简洁的语言写故事本身，接近舞台剧本的实用性：谁在何处，做了什么，说了什么。
   - 不写模板字段和制作说明：不写系列、章节、叙事原则、素材依据、主要地点、主要人物、核心视觉、本集一句话、剧本结构、旁白基调、制作备注。
   - 不把每集硬拆成“开场钩子 / 人物进入 / 核心冲突 / 精神落点 / 结尾钩子”；只有剧情自然需要时才分场。
   - 完成标准：读者能从头到尾理解发生了什么故事、谁推动了动作、关键台词是什么，且看不到 boilerplate。

3. **镜头组**
   - 将故事拆成可用于图片、视频和漫画生成的镜头组。
   - 每个镜头组只承担一个主要叙事任务、动作目标、信息点或情绪转折。
   - 组内可保留 1-6 个镜头节拍，但这些节拍用于说明连续动作，不默认逐条生成。
   - 完成标准：每个镜头组都能独立导出关键帧、视频段落或漫画页，不依赖抽象心理描写。

4. **资产清单**
   - 拆出角色、服装、道具、场景、关键物件、关键帧、漫画页或视频参考图。
   - 标注哪些资产可复用，哪些必须新建，哪些只作为气氛参考。
   - 完成标准：后续制作人员知道先做什么、保存到哪里、给谁复用。

5. **基础资产优先**
   - 先做身份锚点、角色立绘、服装/状态、场景空镜和道具。
   - 关键帧、漫画页、视频片段最后做，因为它们依赖前序资产。
   - 完成标准：所有关键帧依赖的角色、服装、场景、道具都有明确来源。

6. **关键帧 / 漫画页 / 视频提示词**
   - 从已确认剧本、镜头组和基础资产中提取，不反向改写剧情。
   - 完成标准：每个生成项都能说明对应哪个故事段落或镜头组。

7. **生成后质检**
   - 检查身份、服装、空间、动作、物理逻辑、风格、人数、画质和可复用性。
   - 完成标准：不合格图/片有失败原因，不进入后续链路。

## 技能自升级规则

- 每次发生大重写、大重构或用户明确纠正创作规范时，必须把新规则沉淀回相关技能；不要只修改当前文件。
- 如果项目内有 `.art-skills/`，同步更新项目技能副本；全局技能和项目技能有冲突时，以用户本次明确纠正后的项目技能为准。
- 升级技能时要删除或替换旧规则，避免新旧规范并存导致下次继续套错模板。

## 通用资产顺序

推荐顺序：

1. 角色身份锚点：脸部、头部、全身、背面或必要状态。
2. 服装 / 发型 / 状态参考：只在角色会换装或状态复杂时制作。
3. 场景空镜：无主角、无命名关键角色，先建立空间和动线。
4. 道具 / 物件：武器、票据、地图、设备、食物、载具、机关等。
5. 剧情关键帧 / 漫画页 / 视频关键帧：按故事或镜头顺序生成。
6. 补图 / 修正版：只修正明确失败点，不无限重试。

## 连续性质检

每个输出都要检查：

- 人物是否还是同一个人；脸、体型、发型、服装、关键配饰是否一致。
- 人物数量是否符合设定；同一人物是否被重复画成多个个体。
- 场景空间是否支持动作：入口、出口、路径、目标物、遮挡和尺度是否可理解。
- 道具是否符合物理与剧情功能；复杂道具是否能看出连接、受力、方向和操作方式。
- 情绪强度是否匹配剧情阶段，是否过度戏剧化或过于静态。
- 风格是否与项目要求一致，而不是只看内容正确。
- 是否误用了废弃素材、旧设定、无关示例或其他项目风格。

## 交付检查

- [ ] 已明确权威源文档和不可使用的旧材料。
- [ ] 剧本/故事没有被资产计划或提示词反向改写。
- [ ] 镜头组可拍、可画、可生成，且没有退化成逐镜头碎表。
- [ ] 资产清单覆盖角色、场景、道具和关键帧。
- [ ] 基础资产先于关键帧和视频提示词。
- [ ] 每个生成项都有用途、依赖、保存路径和质检标准。
- [ ] 不合格资产不会进入后续参考链路。
- [ ] 未经用户明确要求，没有提交图片或视频生成任务。
