# 图片质量检查

每生成一张图片后、把它当作最终资产或后续参考图之前，立即使用本技能检查。不要连续生成很多张后才统一检查。本检查是闸门：通过、带定向修复重生、或停止并报告阻塞。

## 必备输入

每张图检查前，先收集：

- 生成图片本身。
- 对应的完整任务 prompt 或 TOML `[[tasks]]` 块。
- `target_image_path`、`aspect_ratio` 和任务 `id`。
- 任务列出的所有参考图，并标明各自用途：身份、服装/造型、道具/物件、环境/风格、构图/动作等。
- 项目级美术方向，尤其是媒介类型、风格、色彩、光线、材质、完成度和画面用途。

## 检查顺序

### 1. 硬性完整性闸门

如果图片出现以下任一问题，直接拒收：

- 多余肢体、缺失肢体、重复手脚、手指融合、关节不可能、透视破碎。
- 角色身体、服装、道具、家具或环境结构处在互相矛盾的位置。
- 鞋履、手套、帽子、武器、工具、容器等物件无法物理适配对应身体或动作。
- 角色年龄、身份或物种含混到影响任务要求。
- 裸露、露出内衣、露骨性接触、暴力/血腥等内容超出 prompt 或项目边界。
- 水印、文字覆盖、不可读伪 UI、随机签名，或 prompt 要求无文字时出现可见文字。
- 裁切掉必要的核心动作、角色身份、关键道具或环境叙事线索。

### 2. 角色 / 主体身份检查

把生成主体和对应参考图逐项对比。

角色图重点检查：

- 年龄段、性别表现、体型、脸型、五官、发型、肤色等身份锚点。
- 服装轮廓、配饰、鞋履、发饰、眼镜、帽子、装备等造型锚点。
- 表情和气质是否符合任务描述与角色阶段。
- 身高比例、头身比、体态、站姿/坐姿/动作姿态是否与参考图一致。

非人物主体重点检查：

- 道具、动物、建筑、车辆、植物、食物、UI 或产品的形状、结构、材质和尺度。
- 是否能一眼识别为任务指定主体。
- 是否误混其他参考图中的无关元素。

如果身份、结构、服装/造型、物件形制或年龄/状态偏离参考图，拒收或重生。

### 3. 解剖、物理与接触逻辑检查

所有图片都要检查物理逻辑；人物、动物、手部、足部、服装、道具交互图要重点检查。

确认：

- 肢体数量、左右方向、关节连接和受力关系合理。
- 手、脚、爪、翅、尾、衣料、装备、工具等与身体自然连接。
- 服装、鞋履、盔甲、包袋、首饰等穿戴逻辑成立，没有穿帮或融合。
- 手部或工具的接触点符合动作：拿、递、擦、扣、绑、扶、推、拉、按、切、写、打开等。
- 被拿起、放置、悬挂、支撑或承重的物体有合理方向、重量感和支点。
- 近景局部没有造成头、手、脚、道具之间不可能的几何关系。
- 如果 prompt 限定接触范围或动作边界，画面严格遵守该边界。

即使画面有美感，只要解剖或物理结构不成立，也要拒收。结构逻辑优先于装饰效果。

### 4. Prompt 符合度检查

只对照当前任务的准确 prompt，不用记忆或相邻任务补全。

确认：

- 必要地点、时间、天气、空间类型和环境元素出现。
- 必要人物 / 主体数量正确，没有自动增减重要角色或物件。
- 必要动作、关系、情绪和事件节拍清楚可读。
- 必要道具、服装、装备、场景结构出现，且没有被无关物替代。
- 画幅、景别、构图意图和焦点层级符合任务要求。
- 画面边界内没有出现 prompt 或项目规则不允许的内容。

如果图片很好看但没有表现要求的任务内容，拒收。

### 5. 参考图对齐检查

按参考图用途逐项检查，不让某一张参考图错误主导整张图。

常见参考用途：

- 身份参考：脸、年龄、发型、体型、气质、物种或主体结构。
- 全身 / 造型参考：服装、姿态、比例、配饰、鞋履、装备。
- 道具 / 物件参考：材质、轮廓、结构、颜色、尺寸和使用方式。
- 环境参考：建筑、空间布局、家具、植物、光线、色彩、氛围。
- 风格参考：媒介、笔触、渲染方式、材质处理、对比度、色彩系统。
- 构图 / 动作参考：机位、景别、姿势、动态、人物距离和视觉重心。

如果某张参考图被错误复制，例如只复制了道具却丢失角色身份，或复制了场景却引入无关人物和风格，拒收或降级为局部参考。

### 6. 项目美术与视觉风格检查

所有通过图片必须符合当前项目的美术方向。不要把以下示例当作固定风格；应以当前项目文档和已确认参考图为准。

检查维度：

- 媒介类型：摄影、电影剧照、3D 渲染、2D 插画、2.5D、漫画、水彩、油画、产品摄影等。
- 人物 / 主体比例：写实、半写实、Q 版、夸张漫画、产品精修等是否匹配。
- 材质语言：皮肤、毛发、布料、金属、玻璃、木材、石材、纸张、食物、液体等是否和项目一致。
- 色彩系统：饱和度、色温、对比度、调色倾向是否偏离。
- 光线系统：自然光、棚拍、电影光、舞台光、体积光、低调光、高调光等是否匹配。
- 细节密度：是否过糊、过锐、过脏、过油、过度堆细节或过度简化。
- 完成度：草图、概念图、成片剧照、角色定妆、道具设定、漫画页等是否符合用途。

如果结果虽然精致，但明显偏离项目媒介、材质、色彩、光线或完成度，拒收或降级为气氛参考。

### 7. 构图与可读性检查

确认：

- 主要事件、主体或资产用途能在几秒内读懂。
- 视觉层级清楚：主体身份、动作关系、关键道具、环境信息各有位置。
- 构图服务当前用途：角色图看身份，环境图看空间，道具图看结构，概念剧照看动作和关系，漫画页看叙事顺序。
- 画面没有被无关人物、杂物、装饰、文字或过强背景抢走焦点。
- 主体没有被裁切、遮挡或缩小到影响任务用途。
- 构图没有意外改变剧情权力关系、情绪方向或视觉重点。

### 8. 决策标准

使用以下结果之一：

- `PASS`：保存 / 复制到 `target_image_path`，可进入后续参考链路。
- `PASS_WITH_NOTES`：可用，但记录小瑕疵或使用限制；不得作为高权重身份/结构参考。
- `REGENERATE_MINOR`：只有一个或少数可定向修复的小问题，例如缺道具、环境弱、局部材质不准。
- `REGENERATE_MAJOR`：身份、解剖、物理逻辑、参考对齐、风格或核心任务内容失败。需要重写 prompt 或更换参考图。
- `BLOCKED`：原任务请求被生成器拒绝，或无法安全/稳定生成。保留任务意图，改成合规版本，并说明改动。

如果图片在解剖、物理逻辑、prompt 符合度、关键身份、参考对齐或项目风格上失败，绝不能标为 `PASS`。

## 重生 Prompt 规则

重生时只加入定向修复，不新增剧情。

必须写清：

- 身份锁定：主体是谁，脸/体型/年龄/物种/结构如何保持。
- 造型锁定：服装、发型、配饰、鞋履、装备或产品外观。
- 物理锁定：肢体数量、关节连接、接触点、承重、方向和空间关系。
- 动作锁定：准确的动作、互动、姿态或物件使用方式。
- 场景锁定：地点、空间结构、光线、关键陈设或背景关系。
- 风格锁定：当前项目的媒介、渲染、色彩、材质和完成度。
- 高风险失败项：只列当前失败相关的问题，不堆砌无关负面词。

提示词应以正向修复为主；如果确实需要负面约束，只写会反复导致失败的具体问题。

## 报告格式

每张生成图检查后，记录一条简短质检说明：

```text
[id] CHECK: PASS
- 身份/主体：...
- 结构/物理：...
- Prompt/内容：...
- 参考图：...
- 风格：...
- 保存：<path>
```

或：

```text
[id] CHECK: REGENERATE_MAJOR
- 失败点：...
- 修复 prompt：...
```

说明要简洁，但必须具体到后续审查者能看懂为什么通过或为什么重生。
