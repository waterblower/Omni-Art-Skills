# Hermes 全能编导

[English](README.md)

这是一个面向 Hermes Agent 与 Codex 的生产型创作技能库。它不是提示词合集，而是把 Agent 组织成兼具制片、导演、美术指导、资产管理与视觉质检能力的创作协作者。

整套技能按依赖关系组织：先确认故事与权威资料，再形成镜头组，建立可复用资产与参考图，编排可执行的图片或视频任务，最后在图片进入后续链路前完成质量复核。

## 技能地图

| 阶段 | 技能 | 主要产物 |
| --- | --- | --- |
| 制作基础 | `creative-production-pipeline` | 有阶段门禁的全流程与资产顺序。 |
| 故事与镜头 | `script-to-shot-table`、`manga-creation-pipeline` | 可生成的镜头组、漫画故事与漫画页。 |
| 资产与美术 | `character-reference-pipeline`、`image-art-direction` | 可复用角色资产与经过审查的视觉参考。 |
| 生成编排 | `image-generation-plan`、`video-prompt-director` | 逐图 TOML 计划与可执行视频提示词。 |
| 质量与风格 | `image-quality-check`、`style-distill`、`cover-image` | 通用图片质检、可复用风格提取与中文视频封面。 |

## 技能说明

### 1. 制作基础

#### [creative-production-pipeline](creative-production-pipeline/SKILL.md) — 总制片 / 总导演

当输入是故事梗概、世界观、分集大纲、广告创意或其他零散创作资料时，用它建立全局制作流程。它从权威源文档确认开始，依次管理剧本、镜头组、资产、图片/视频生产与生成后质检。

- 规定故事、镜头、资产与生成执行的先后关系。
- 要求角色身份锚点、场景、道具、服装等基础资产先于概念剧照、漫画页和视频生成。
- 保护剧情与连续性：图片计划和提示词不能凭空加戏、改变关系，失败资产不能进入下游参考链路。
- 它回答“下一步该产出什么”，再把具体工作交给专项技能。

### 2. 故事、镜头组与漫画

#### [script-to-shot-table](script-to-shot-table/SKILL.md) — 剧本转镜头组导演

把剧本、梗概、小说段落、口播稿、广告脚本或聊天式剧情说明，转成服务 AI 图片、视频和漫画的 Markdown 镜头组文档。

- 默认生产单位是镜头组，而不是被拆得过碎的逐镜头表。
- 每组承担一个明确叙事任务，只保留解释连续动作所需的少量节拍。
- 支持极简格式：编号、建议时长、场景、人物、内容；只有用户要求导演说明书时才扩展字段。
- 是图片计划、视频提示词、概念剧照与漫画页的直接上游。

#### [manga-creation-pipeline](manga-creation-pipeline/SKILL.md) — 漫画编剧、导演与页面规划

从世界观、故事方向或角色设定出发，开发可阅读的漫画章节和可生成的漫画页面。

- 区分稳定世界观、具体章节故事与页面生成材料，避免把设定文档、剧情稿和提示词混成一份。
- 协助提出可选漫画方向，组织章节升级、高潮、后果、页尾钩子与信息释放控制。
- 一组镜头对应一张漫画图或一页漫画：单幅大图、多格页、动作页、蒙太奇页或情绪页。
- 规定手机阅读下的分格数、画幅、阅读顺序、版面层级与提示词写法。

#### [video-prompt-director](video-prompt-director/SKILL.md) — 视频提示词导演

把已经确认的镜头组或场景段落，转成简洁、可视、可听、可执行的视频生成提示词。

- 提示词按“环境 → 人物位置关系 → 连续动作 → 台词时机 → 有动机的镜头运动 → 结束画面 → 声音”组织。
- 用片段时长约束动作容量，避免短片段里塞进无法稳定生成的一长串事件。
- 要求实际读图，并为每张参考图设定单一、清楚的使用职责。
- 台词必须明确说话者、语言、原句和发生时机；默认不提交视频生成任务。

### 3. 资产流水线与美术指导

#### [character-reference-pipeline](character-reference-pipeline/SKILL.md) — 角色资产入库

把一次性角色图、群像、临时设定或文字描述，标准化为可复用的角色参考资产，减少后续身份漂移。

- 按角色重要性决定需要的正脸、后脑勺、正背立绘、服装图与状态图，避免无效堆图。
- 建立唯一、干净的源头脸作为身份锚点；之后任何带脸图片只能从它锁定身份。
- 把脸部身份资产与服装/状态参考分离，避免服装图污染角色长相。
- 提供背景、构图、文件命名、浅层目录与验收标准，适合长期项目资产库。

#### [image-art-direction](image-art-direction/SKILL.md) — 图片美术指导 / 参考图审查

当重点是看图、选图、审图、判断一张图能否进入项目参考链路时，使用本技能，而不是直接写 TOML。

- 必须实际读图，不能仅凭文件名猜测图片内容。
- 按具体用途把图判为主参考、局部参考、气氛参考或不可用。
- 区分“内容正确”“画风兼容”“适合进入下游链路”三件不同的事。
- 审查构图、媒介与材质、人物连续性、空间可信度和常见生成问题；可以建议重生或替换，但不直接改写计划。

### 4. 图片编排与质量门禁

#### [image-generation-plan](image-generation-plan/SKILL.md) — 图片制片 / TOML 任务编排

基于已有镜头组、制作包和已审核资产，编写每张图片都可独立执行的 TOML 生成计划。

- 强制前置条件：没有镜头组时，不从剧本或大纲直接跳到图片计划。
- 任务顺序固定为：环境/场景 → 道具/工具/服装 → 人物 → 概念剧照。
- 每项任务都有 ID、保存路径、参考图、画幅、提示词及 `# todo`/`# done` 状态。
- 要求提示词自包含、路径相对、参考图按图号与用途明确声明；概念剧照必须从镜头组派生，不是任意“关键帧”。
- 还规定项目级人物/道具复用、计划校验和 TOML 内的连续性约束。

#### [image-quality-check](image-quality-check/SKILL.md) — 通用图片质量闸门

判断生成图片是否可接收、可交付或可作为下游参考。它刻意不承担纯画风相似度判断。

- 检查文件可读性、尺寸/画幅、透明通道、任务符合度、主体与身份、解剖、物理接触、构图和常见生成伪影。
- 返回结构化状态：`PASS`、`PASS_WITH_NOTES`、`REGENERATE_MINOR`、`REGENERATE_MAJOR` 或 `BLOCKED`。
- 只输出明确的修复指令，不生成图片、不保存文件、不直接修改图片计划。
- 对风格测试图，只在通用质量通过后交给专门的风格匹配阶段，避免把技术质检和画风评判混为一谈。

### 5. 风格与发布物料

#### [style-distill](.agents/skills/style-distill-v1.4/SKILL.md) — 可复用美术风格提取与迭代

当用户要求获取、提取、萃取、蒸馏或反推参考图画风时，启动完整的风格蒸馏流水线。

- 先判定参考图媒介类型：纯 2D、纯 3D 渲染、2.5D、2D/3D 混合或真实摄影。
- 将能迁移的风格不变量与角色、道具、场景等参考图内容分离。
- 必须真实生成并读图比较：至少两轮、每轮四类跨内容测试图，之后再生成十六张独立材质/纹理锚点图。
- 所有门槛通过后才落盘一个全新的可复用风格生成技能目录；它不是“只给一段 prompt”的轻量技能。

#### [cover-image](.agents/skills/cover-image/SKILL.md) — 中文视频封面与缩略图

基于用户提供的参考图制作中文视频封面，尤其适用于 AI 与技术教程中需要准确中文标题的场景。

- 保留参考图的柔和光线和精细 3D 人像质感，并保证文字不遮挡脸部。
- 除非用户指定单一规格，默认输出 16:9、4:3、3:4 三种比例。
- 必须核验中文标题逐字准确；模型文字不准时，使用确定性本地排版方式修正。
- 默认在对话中交付；只有用户明确给出技能目录外的保存路径时才写入文件。

## 推荐使用顺序

### 影视、短剧、AI 短片

1. 用 `creative-production-pipeline` 确定权威资料和生产门禁。
2. 完成剧本或故事，再用 `script-to-shot-table` 得到镜头组。
3. 新增长期角色时用 `character-reference-pipeline`；审查场景、道具和外部参考时用 `image-art-direction`。
4. 基于确认后的镜头组，用 `image-generation-plan` 编写资产优先的 TOML 任务。
5. 基于镜头组，用 `video-prompt-director` 写视频片段提示词。
6. 图片成为交付物或下游参考前，用 `image-quality-check` 复核。

### 漫画

1. 用 `manga-creation-pipeline` 开发方向、章节故事、信息释放计划和页面镜头组。
2. 页面生成前先建立并审核角色参考资产。
3. 由确认后的漫画镜头组派生页面任务，再进入质量复核。

### 风格项目与教程发布

1. 用 `style-distill` 把参考画风变成经过测试的可复用技能。
2. 用制作与图片计划技能应用该风格，但不复制参考图的具体内容。
3. 用 `cover-image` 制作最终中文视频封面或缩略图。
