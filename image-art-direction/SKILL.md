# 图片美术风格导演

## 适用范围

当任务涉及选择参考图、分析图片风格、制定图片生成提示词、审核生成图、统一角色/环境/道具/关键帧风格时，使用本技能。

本技能只沉淀通用方法，不指定任何项目专属画风。具体项目可以是写实、动画、漫画、3D、摄影、插画、产品摄影或其他风格，但必须从项目源文档和已确认参考图中读取，而不是凭空套用。

## 核心原则

1. **内容可用 ≠ 风格可用。** 构图、主体、景别正确，不代表它适合项目。
2. **实际读图，不看文件名下结论。** 选择参考图前必须观察图片内部信息。
3. **先找文本美术规格，再用图片验证。** 项目文档的明确美术要求优先；高质量项目内图片用于校准。
4. **文本与图片冲突时先报告。** 不要擅自生图或把冲突合成新风格。
5. **不合格参考图不得继续绑定。** 画质、清晰度、风格、材质、比例、光影不合格时，先说明问题并替换或补图。
6. **负面约束要精简。** 避免项只写高风险问题，不堆砌无关禁词。

## 生图前流程

1. **明确用途**
   - 角色参考、环境参考、道具参考、动作参考、食物参考、分镜补图、海报概念、关键帧、漫画页或其他用途。
   - 完成标准：图片用途能决定画幅、构图、背景、是否需要人物和参考图。

2. **查找美术风格文本**
   - 阅读设定文档、剧本、角色设定、世界观、README、prompt 文档、制作包等。
   - 重点提取：媒介、渲染方式、材质、光影、角色比例、色彩、镜头质感、明确不要项。
   - 完成标准：能写出一条项目主风格句和 3-5 个风格检查点。

3. **实际读入相关图片**
   - 角色图：判断比例、脸、发型、服装、材质、线条或渲染语言。
   - 环境图：判断建筑、地形、植物、水面、金属、烟雾、光影和细节密度。
   - 道具图：判断结构、材质、比例和可复用性。
   - 完成标准：知道哪些图能做身份锚点，哪些只能做气氛参考。

4. **审查候选图**
   - 内容是否支持镜头表达。
   - 景别和构图是否适合当前用途。
   - 主体是否清晰，画质是否足够。
   - 风格是否和项目主风格一致。
   - 是否能与其他项目图放在一起。
   - 完成标准：候选图被标为“可用 / 只作气氛参考 / 不可用”，并说明原因。

5. **写提示词**
   - 先写风格基准，再写具体内容。
   - 参考图关系必须明确：锁身份、锁服装、锁场景、锁道具、锁构图、锁情绪或只参考风格。
   - 完成标准：提示词能直接交给图像模型，不只是剧情摘要。

6. **生成后复核**
   - 实际打开新图检查，不只看文件名或工具返回。
   - 只有内容、画质、清晰度和风格都合格，才进入后续链路。

## 图片提示词推荐顺序

```text
用途与画幅 → 项目主美术风格 → 主体/场景/道具内容 → 景别与构图 → 材质/光影/色彩 → 参考图绑定关系 → 必须避开的关键问题 → 无文字/无水印等输出约束
```

## 历史场景 / 室内空镜提示词注意事项

写古代、历史、寺院、译场、宫殿、书房、课堂等室内场景提示词时，必须主动约束以下问题：

- **陈设密度**：室内成列经案、蒲团、书架、灯烛、器具不要过密；要留出真实通道、坐卧空间和视觉呼吸感，避免像仓库、展厅或复制粘贴的阵列。
- **柱子数量**：柱子、梁架、帷幔、屏风等结构件只写必要数量和空间逻辑；除非源文档明确要求密集柱廊，否则在避免项中写“不要过多柱子、不要重复柱阵”。
- **时代污染**：古代场景即使是远景/背景人物，也必须排除现代人特征；避免现代发型、现代服装、现代鞋包、手机、相机、口罩、游客姿态、现代灯具、电线、标牌、玻璃幕墙、现代路面和现代陈设。
- **空镜人物控制**：如果是“空镜”，默认无人物；若需要极少量远景僧人/侍者/工匠/行人，必须写清身份、服饰时代、数量上限和低存在感。

## 常见风格质检维度

- 媒介是否正确：摄影、电影剧照、3D 渲染、漫画、插画、产品摄影等。
- 清晰度是否足够：主体、脸、手、道具结构、空间边界是否可读。
- 材质是否可信：皮肤、布料、金属、玻璃、水、木、石等是否区分清楚。
- 光线是否符合项目：自然光、棚拍、电影光、体积光、低调光等是否一致。
- 色彩是否统一：色温、饱和度、对比度、调色倾向是否偏离。
- 细节密度是否合适：是否过糊、过脏、过油、过锐、过度堆细节。
- 室内陈设密度是否可信：桌案、蒲团、灯烛、书架、器具是否过度成列、过密、缺少通道和呼吸感。
- 建筑结构件是否合理：柱子、梁架、屏风、帷幔是否数量过多、重复生成、破坏空间真实感。
- 历史/古代场景是否有时代污染：远处人物是否像现代人，是否混入手机、相机、现代服装鞋包、现代灯具、电线、标牌、玻璃幕墙等。
- 构图是否服务用途：角色图干净，场景图看空间，道具图看结构，关键帧看动作目标。

## 候选图报告格式

```text
这张图内容上可用，但美术风格不适合：问题是 <具体风格偏差>。它可以作为 <气氛/构图/道具> 参考，但不能作为 <身份/主风格/关键帧> 参考。建议重生成或替换为 <目标风格与用途> 的版本。
```

## 交付前检查

- [ ] 是否查过项目内美术风格文本。
- [ ] 是否实际读入候选参考图。
- [ ] 是否区分了内容可用和风格可用。
- [ ] 是否明确了每张参考图的用途。
- [ ] 是否剔除或报告了不合格参考图。
- [ ] 提示词是否先写风格，再写具体内容。
- [ ] 避免项是否短而具体。
- [ ] 生成后是否再次读图质检。
- [ ] 未经用户明确要求，没有提交生成任务。
