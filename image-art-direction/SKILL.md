# 图片美术指导 / 视觉质检

## 适用范围

当任务核心是“看图、选图、审图、判断图能不能作为参考或成品继续使用”时，使用本技能。

典型场景：

- 从多个已有图片中选择最适合当前项目画风的参考图。
- 判断某张参考图能否作为身份锚点、服装参考、场景参考、道具参考、构图参考或仅作气氛参考。
- 审查生成结果是否符合项目画风、内容、清晰度、构图和连续性要求。
- 用户问“这张图能不能用 / 哪张更好 / 为什么不对 / 要不要重生”。
- 在图片计划引用已有图之前，实际读图确认图像内容和风格是否匹配。

不适用场景：

- 写 TOML 图片生成计划、改 `reference_image_paths`、排任务顺序、标 `# todo/# done`：使用 `image-generation-plan`。
- 用户要求 extract / distill / 萃取 / 反推画风，并需要迭代生成候选图：使用 `style-distill`。
- 建立角色源头脸、正反立绘、服装状态图资产流水线：使用 `character-reference-pipeline`。

## 核心原则

1. **必须实际读图。** 不用文件名、路径或任务标题直接判断图片内容。
2. **内容可用不等于风格可用。** 构图、主体正确的图，如果媒介、材质、色彩或完成度偏离项目，也不能直接作为主参考。
3. **参考用途必须分级。** 同一张图可能适合锁场景结构，但不适合锁画风；可能适合气氛，但不适合作为角色身份锚点。
4. **不合格图不得进入关键链路。** 身份错误、风格偏差大、画质糊、结构错误或尺度不稳的图，应标为不可用或只作低权重气氛参考。
5. **项目内已确认图优先。** 外部图、临时图、未审核图只能在明确说明用途和风险后使用。
6. **非写实项目优先 styled 图。** 在当前项目画风不是写实照片风格时，选择既有参考图应优先考虑文件名带 `styled` 前缀或后缀的版本，但仍需读图确认。
7. **只给视觉判断，不篡改计划。** 如果需要修改 TOML 或重新编排任务，切换到 `image-generation-plan` 规则执行。

## 工作流程

### 1. 明确审查目标

先判断当前要审的图用于什么：

- 角色身份 / 脸部锚点
- 全身比例 / 服装 / 发型 / 姿态
- 鞋履 / 道具 / 物件结构
- 场景空间 / 环境氛围 / 光线
- 概念剧照 / 情绪 / 动作关系
- 纯美术风格参考
- 成品图质量验收

不同用途的合格标准不同；不要用“好看”替代用途判断。

### 2. 实际读图并记录可见事实

读图时先写可观察事实，不先下结论：

- 主体：画面中有哪些人、物、场景。
- 构图：景别、机位、主体占比、视觉焦点、留白。
- 风格：媒介类型、2D/3D/摄影倾向、笔触、材质、色彩、对比度、光影。
- 质量：清晰度、脸/手/脚/道具结构、边缘、噪点、畸变、重复肢体。
- 连续性：角色是否像既定身份，服装/鞋履/道具/场景是否和项目一致。

### 3. 判定参考用途等级

使用以下等级输出结论：

- **可用作主参考**：内容、风格、画质、连续性都匹配；可直接进入后续任务参考链路。
- **可用作局部参考**：只适合锁某一部分，例如鞋履形制、场景布局、光线、构图；prompt 必须明确“只参考 X”。
- **只作气氛参考**：风格或内容有偏差，只能低权重参考色调、氛围或大致空间感，不能锁身份/服装/关键造型。
- **不可用**：身份错误、风格不符、画质不合格、结构错误、尺度错误、内容污染或与源文档冲突。

### 4. 对比项目风格

检查候选图是否能和项目已确认图片放在同一个视觉系统里：

- 媒介是否一致：摄影、3D 渲染、2D 插画、2.5D、漫画等。
- 渲染与材质是否一致：皮肤、布料、金属、玻璃、木材、石材、头发、鞋履。
- 色彩和对比度是否一致：饱和度、色温、阴影浓度、调色倾向。
- 细节密度是否一致：是否过糊、过锐、过脏、过油、过度堆细节。
- 人物比例和脸部语言是否一致：是否像同一项目的人物系统。
- 场景空间是否可信：室内陈设、通道、尺度、结构件是否合理。

### 5. 生成后质检

对于已生成图片，至少检查：

- 是否符合当前任务 prompt 的主体、人数、动作、道具和地点。
- 是否保持角色身份、年龄、脸、发型、服装、鞋履和体态连续性。
- 是否有多手、多脚、重复人物、错肢、糊脸、错鞋、错道具、文字水印等问题。
- 是否符合项目美术风格，而不是只“看起来精致”。
- 是否能作为后续参考继续使用；不能用时说明重生原因。

## 历史 / 古典 / 室内场景审查重点

审查古代、历史、寺院、译场、宫殿、书房、课堂、豪宅等室内场景时，重点看：

- **陈设密度**：桌案、书架、灯烛、器具是否过密，是否留出真实通道和行动空间。
- **结构件数量**：柱子、梁架、帷幔、屏风是否过多或重复，空间是否像展厅/仓库。
- **时代污染**：是否混入现代发型、现代服装鞋包、手机、相机、口罩、现代灯具、电线、标牌、玻璃幕墙等。
- **人物控制**：空镜是否真的空；如果有远景人物，身份、服饰和存在感是否符合要求。
- **可执行空间**：后续人物互动、走位、坐卧、开门、递物是否有足够空间。

## 输出格式

### 单图审查

```text
结论：可用作主参考 / 可用作局部参考 / 只作气氛参考 / 不可用
用途：<身份 / 服装 / 场景 / 道具 / 构图 / 氛围 / 成品质检>
可用点：<具体可见优点>
风险或问题：<具体偏差，不写空泛评价>
处理建议：<沿用 / 降级为气氛参考 / 替换 / 重生 / 需要补哪类图>
```

### 多图选择

```text
推荐：<文件 A>
原因：<为什么更符合项目风格和当前用途>
不选：<文件 B/C>
原因：<具体问题或只能低权重参考的范围>
后续使用方式：<在 prompt 中应如何声明参考用途>
```

## 交付前检查

- [ ] 是否实际读入并观察图片内容。
- [ ] 是否区分了内容可用、风格可用和链路可用。
- [ ] 是否明确每张图的参考用途等级。
- [ ] 是否指出不合格图的具体问题，而不是只说“不好”。
- [ ] 是否避免把视觉 QA 规则和 TOML 编排规则混在一起。
- [ ] 如果需要修改图片计划，是否切换到 `image-generation-plan` 的规则执行。
- [ ] 如果需要风格萃取和迭代生成，是否切换到 `style-distill`。
