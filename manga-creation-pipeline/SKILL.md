# 漫画创作与漫画页生成流程

## 适用范围

当用户需要开发漫画方向、写漫画章节、把故事拆成漫画镜头组、规划漫画页图片生成、审查漫画分格页时，使用本技能。

本技能只保留通用漫画创作和漫画图生产方法，不绑定任何具体项目、画风、角色或世界观。

## 核心区分

漫画开发中要分清三类内容：

1. **世界观知识**
   - 稳定规则、生态、势力、物种、技术或魔法体系。
   - 适合放在设定文档或项目圣经中。

2. **具体故事材料**
   - 主角弧光、章节事件、人物过去、冲突、章节节拍。
   - 适合放在漫画章节、故事稿或系列方向文档中。

3. **镜头组 / 页面生成材料**
   - 一组等于一张要生成的漫画页或图像。
   - 写清页面构图、分格数量、视觉重点、可选台词和避免混淆点。

不要把世界观文档写成剧情草稿，也不要把长篇故事原文直接丢给图片模型。

## 漫画方向头脑风暴

当用户要漫画方向时：

1. 先读取已有设定和用户偏好，不凭空换题材。
2. 给出 2-3 个可执行方向，而不是散乱点子堆。
3. 每个方向包含：
   - 标题或标题候选。
   - 一句话核心。
   - 主角视角。
   - 表层钩子。
   - 深层主题或长期谜题。
   - 为什么适合漫画表达。
   - 第一话钩子。
4. 合并功能重复的方向，避免多个选项只是换皮。
5. 给出推荐：最稳测试、最适合长篇、最有差异化。

完成标准：用户能从输出中直接选择一个方向开始写第一话。

## 漫画章节故事结构

写 `故事.md` 或章节故事稿时，写可阅读的漫画故事，不只写要点。

常用第一话结构：

1. **开场奇观**：用一个强画面卖出世界或题材。
2. **日常对比**：展示主角和环境的差异。
3. **谜题物件 / 问题**：地图、信物、标记、遗迹、异常现象等。
4. **引发事件**：外来者出现、禁忌被触碰、任务降临、怪物出现。
5. **功能性闪回**：只在能解释行为或危险时使用，避免大段设定灌输。
6. **升级**：新威胁比旧威胁更强或性质不同。
7. **核心情绪认知**：恐惧、羞耻、好奇、欲望、责任或选择。
8. **能力 / 决断时刻**：通过问题解决呈现，不做无缘由开挂。
9. **后果**：不解决所有问题，要显示世界改变。
10. **出发 / 下一目标**：主角选择行动。
11. **页尾钩子**：追兵、报告、神秘线索、下一个目的地或反转。

完成标准：章节能从头读到尾，有开场、升级、高潮、后果和钩子。

## 信息释放控制

第一话或新篇章不要解释所有谜团。建议在故事或规划后加入：

```markdown
## 本话信息释放控制

本话明确：
- ...

暂不解释：
- ...
```

作用：保护长期谜题，避免设定说明挤压漫画节奏。

## 镜头组工作流

不要把长篇故事直接交给图片生成。先拆成镜头组。

### 定义

一个 **镜头组** 等于一张生成图。它可以是：

- 单幅大图。
- 多格漫画页。
- 动作页。
- 蒙太奇页。
- 安静情绪页。
- 结尾钩子页。

### 什么时候用单幅大图

适用于：

- 建立壮观地点或时空关系。
- 重要角色登场。
- 觉醒、宣言或标志性瞬间。
- 章节结尾海报式钩子。
- 重要情绪停顿。

### 什么时候用多格漫画页

适用于：

- 普通动作连续性。
- 因果动作。
- 对话交换。
- 调查、阅读、发现线索。
- 喜剧节奏。
- 角色注意到多个细节。

### 每个镜头组字段

```markdown
## 镜头组 NN｜<短标题>

- 图像类型：单幅大图 / 多格漫画页 / 动作页 / 情绪页 / 蒙太奇页
- 建议画幅：竖版 / 横向宽幅 / 跨页 / 2:3 / 3:4 / 其他
- 对应故事：<覆盖哪个故事节拍>

画面内容：
<单幅大图写这里>

分格设计：
<多格页按 1、2、3... 写清每格>

可加台词：
<可选对白、旁白、拟声词；不需要则写无>

画面重点：
- <生成必须保留什么>
- <不要混淆什么>
```

完成标准：人类或图片模型能独立生成每个镜头组，不需要从长篇故事里猜分格边界。

## 手机阅读漫画页规则

- 每页最多 6 格；需要文字、表情或动作可读时，优先 3-5 格。
- 默认使用竖版漫画比例，如 2:3 或 3:4；除非用户明确要求，不把漫画页默认做成超长短视频比例。
- 每个漫画页生成块都写明画幅比例。
- 对话气泡可以有，但文字要大、少、清楚。
- 画面用构图、表情、动作和光影讲故事，不依赖解释性旁白框。

## 漫画分格版面规则

漫画页必须像完成漫画页，而不是均匀故事板网格：

- 不要所有格子等宽等高。
- 混合大锚点格、窄竖格、小反应特写、斜切格、插入格、静默情绪格、出血格或无边框格。
- 阅读顺序清楚，默认从左到右、从上到下，或按项目指定阅读方向。
- 一个关键动作或情绪支配页面，辅助格解释原因、细节和反应。
- 冲击、追逐、碰撞、觉醒、恐怖或转折可用斜切/破边框。
- 观察、证据、食物、地图、道具和情绪停顿可用安静矩形格。
- 允许局部元素越过格线，但不能牺牲可读性。

## 漫画图提示词结构

```text
输出目的和画幅 → 漫画/美术风格基准 → 角色/环境/道具内容 → 分格与阅读顺序 → 光线/色彩/材质 → 角色连续性锚点 → 避免项
```

注意：

- 角色参考图先于漫画页。
- 漫画页优先清晰叙事，而不是把多个漂亮画面随意拼贴。
- 文字只限对白气泡、内心独白气泡、剧情必要标牌/道具文字、拟声词。
- 避免解释性旁白框替代画面表达。
- 不要要求模型生成大段可读文字；最终文字可由后期单独排版。

## 审查清单

- [ ] 世界观和具体故事没有混写到同一个无边界文档。
- [ ] 故事有开场、升级、高潮、后果和钩子。
- [ ] 有信息释放控制，未过早解释所有谜团。
- [ ] 镜头组一组对应一张生成图/漫画页。
- [ ] 每组有图像类型、画幅、对应故事、画面内容、分格设计和画面重点。
- [ ] 大图页和多格叙事页比例平衡。
- [ ] 漫画页不是等宽等高故事板网格。
- [ ] 分格阅读顺序清楚。
- [ ] 台词简短可读，气泡不遮挡关键动作或面部。
- [ ] 角色参考图已先于漫画页建立。
- [ ] 不提交图片生成任务，除非用户明确要求。
